Adresse bearbeiten

Hilfe > Adresse bearbeiten

Übersicht

Der Adresse bearbeiten Dialog ist ein universeller CRUD-Dialog für die Verwaltung von Adressen in HCC Plan. Er wird in verschiedenen Modulen verwendet und bietet vollständige Funktionalität zum Erstellen, Bearbeiten und Löschen von Adressen.

Hauptfunktionen

  • CRUD-Operationen: Erstellen, Bearbeiten und Löschen von Adressen in einem einzigen Dialog
  • Automatische Modus-Erkennung: Create-Modus oder Edit-Modus je nach Aufruf
  • Vollständige Adressdaten: Straße, Postleitzahl, Stadt und optionale Bezeichnung
  • Eingabevalidierung: Automatische Überprüfung aller Pflichtfelder
  • Command-Integration: Undo/Redo-Funktionalität über das Command-System
  • Signal-Kommunikation: Benachrichtigung anderer Module über Adressänderungen
  • Multi-Modul-Verwendung: Gemeinsame Nutzung in verschiedenen Bereichen der Anwendung

Grundbedienung

  1. Dialog öffnen: Über entsprechende Schaltflächen in verschiedenen Modulen
  2. Adressdaten eingeben: Füllen Sie die Pflichtfelder Straße, PLZ und Stadt aus
  3. Bezeichnung hinzufügen: Optionale beschreibende Bezeichnung eingeben (z.B. "Klinik XYZ")
  4. Speichern: Klicken Sie auf "Speichern" oder drücken Sie Enter
  5. Validierung: System überprüft automatisch die Vollständigkeit der Eingaben

Tastaturkürzel

F1: Diese Hilfe-Seite öffnen

Enter: Adresse speichern (in jedem Eingabefeld)

Escape: Dialog abbrechen

Tab: Zwischen Eingabefeldern wechseln

Verwandte Funktionen

  • Mitarbeiter-Events: Für Event-Standorte und Teilnehmer-Adressen
  • Stammdaten: Zentrale Verwaltung aller Stammdaten inklusive Adressen
  • Projekt-Einstellungen: Für Standard-Arbeitsplatz-Adressen

Anwendungsbereich

Der Adresse bearbeiten Dialog wird in verschiedenen Bereichen von HCC Plan verwendet:

  • Employee Event Management: Erfassung von Event-Standorten und Veranstaltungsorten
  • Person Management: Verwaltung von Mitarbeiter-Wohnadressen für Fahrtkosten und Erreichbarkeit
  • LocationOfWork Management: Definition von Arbeitsplatz-Standorten für Einsatzplanung
  • Projekt-Setup: Hauptstandort-Definition für neue Projekte

Benutzeroberfläche

Eingabefelder

  • Straße: Vollständige Straßenangabe mit Hausnummer (Pflichtfeld)
  • Postleitzahl: Numerische PLZ für Standort-Zuordnung (Pflichtfeld)
  • Stadt: Städtename für geografische Zuordnung (Pflichtfeld)
  • Bezeichnung: Optionale beschreibende Bezeichnung für bessere Identifizierung

Aktionsschaltflächen

  • "Speichern": Erstellt neue Adresse oder aktualisiert bestehende Adresse
  • "Löschen": Entfernt bestehende Adresse (nur im Edit-Modus sichtbar)
  • "Abbrechen": Schließt Dialog ohne Änderungen zu speichern

Praktische Anwendungsfälle

🎯 Event-Standort erfassen

  1. Dialog über Employee Event Management öffnen
  2. Veranstaltungsort-Adresse eingeben (Straße, PLZ, Stadt)
  3. Bezeichnung wie "Konferenzzentrum München" hinzufügen
  4. Speichern für Verwendung in Event-Planung

🏢 Arbeitsplatz-Standort definieren

  1. Dialog über LocationOfWork-Verwaltung öffnen
  2. Büro- oder Arbeitsplatz-Adresse erfassen
  3. Bezeichnung wie "Hauptbüro Berlin" vergeben
  4. Integration in Einsatzplanung aktivieren

👥 Mitarbeiter-Adresse bearbeiten

  1. Bestehende Adresse über Person Management auswählen
  2. Adressdaten aktualisieren (Umzug, Korrekturen)
  3. Änderungen speichern
  4. Automatische Aktualisierung in allen verknüpften Modulen

Dialog-Modi im Detail

Create-Modus (Neue Adresse)

  • Titel: "Neue Adresse erstellen"
  • Leere Felder: Alle Eingabefelder sind leer und bereit zur Eingabe
  • Löschen-Button: Nicht sichtbar (da noch keine Adresse vorhanden)
  • Speichern: Erstellt neue Adresse in der Datenbank

Edit-Modus (Bestehende Adresse)

  • Titel: "Adresse bearbeiten"
  • Vorausgefüllte Felder: Alle vorhandenen Adressdaten werden geladen
  • Löschen-Button: Sichtbar für Löschoperationen
  • Speichern: Aktualisiert bestehende Adresse in der Datenbank

Häufige Probleme & Lösungen

Problem: "Bitte geben Sie eine Straße ein"

Ursache: Das Straßenfeld ist leer oder enthält nur Leerzeichen.

Lösung: Geben Sie eine vollständige Straßenangabe mit Hausnummer ein.

Problem: "Adresse konnte nicht geladen werden"

Ursache: Die angeforderte Adresse existiert nicht mehr oder Datenbankfehler.

Lösung: Überprüfen Sie die Datenbank-Verbindung und starten Sie den Dialog erneut.

Problem: "Adresse konnte nicht gespeichert werden"

Ursache: Datenbankfehler oder Konflikt mit bestehenden Einträgen.

Lösung: Prüfen Sie auf doppelte Adressen und versuchen Sie es erneut.

Problem: Löschen-Button nicht sichtbar

Ursache: Dialog wurde im Create-Modus geöffnet.

Lösung: Öffnen Sie den Dialog über "Bearbeiten" einer bestehenden Adresse.

Best Practices

💡 Effiziente Adresseingabe

Verwenden Sie die Tab-Taste, um schnell zwischen den Feldern zu wechseln. Enter in jedem Feld speichert direkt die Adresse.

🎯 Sinnvolle Bezeichnungen

Nutzen Sie das Bezeichnungsfeld für eindeutige Namen wie "Hauptbüro München" oder "Konferenzzentrum Nord" zur besseren Identifizierung.

🚀 Command-System nutzen

Alle Adressänderungen werden über das Command-System abgewickelt und unterstützen automatisch Undo/Redo-Operationen.

🔄 Multi-Modul-Integration

Einmal erstellte Adressen können in verschiedenen Modulen wiederverwendet werden - von Event-Management bis zur Einsatzplanung.

Erweiterte Funktionen

Command-System Integration

Der Dialog ist vollständig in das Command-System integriert:

  • Undo/Redo-Unterstützung: Alle Operationen können rückgängig gemacht werden
  • Transaction-Safety: Datenbankoperationen sind atomar und sicher
  • Command-Controller: Zentrale Verwaltung aller Adress-Operationen

Signal-Kommunikation

Der Dialog kommuniziert über Signale mit anderen Komponenten:

  • address_saved: Wird gesendet nach erfolgreichem Erstellen/Aktualisieren
  • address_deleted: Wird gesendet nach erfolgreichem Löschen
  • Automatische Updates: Andere Module werden automatisch über Änderungen informiert

Convenience-Funktionen

Zwei Hilfsfunktionen erleichtern die Verwendung:

  • create_address_dialog(): Schneller Aufruf für neue Adressen
  • edit_address_dialog(): Direkter Aufruf mit bestehender Adress-ID
  • Rückgabewert: Beide Funktionen geben die UUID der bearbeiteten Adresse zurück

Workflow-Beispiele

🏗️ Neue Event-Location erstellen

  1. Dialog über Employee Event Management öffnen
  2. In Straße: "Musterstraße 123" eingeben
  3. Postleitzahl: "80331" eingeben
  4. Stadt: "München" eingeben
  5. Bezeichnung: "Konferenzzentrum Maximilianstraße" eingeben
  6. "Speichern" klicken
  7. Neue Adresse ist verfügbar für Event-Zuordnung

✏️ Bestehende Arbeitsplatz-Adresse aktualisieren

  1. Adresse über LocationOfWork-Verwaltung auswählen
  2. Dialog öffnet im Edit-Modus mit vorausgefüllten Daten
  3. Gewünschte Felder ändern (z.B. neue Hausnummer)
  4. "Speichern" klicken für Update
  5. Alle verknüpften Module erhalten automatisch die Aktualisierung

🗑️ Nicht mehr benötigte Adresse löschen

  1. Adresse im Edit-Modus öffnen
  2. "Löschen" Button klicken
  3. Bestätigungsdialog mit Adress-Details prüfen
  4. "Ja" für endgültiges Löschen bestätigen
  5. Adresse wird aus allen Modulen entfernt

Bedienelemente im Detail

Eingabefelder

  • Straße (Pflichtfeld): Vollständige Straßenangabe mit Hausnummer, z.B. "Musterstraße 123"
  • Postleitzahl (Pflichtfeld): Numerische PLZ für geografische Zuordnung, z.B. "80331"
  • Stadt (Pflichtfeld): Städtename für Standort-Identifikation, z.B. "München"
  • Bezeichnung (Optional): Beschreibender Name für bessere Erkennbarkeit, z.B. "Hauptbüro"

Aktionsschaltflächen

  • "Speichern" (Blau): Standard-Button, erstellt neue oder aktualisiert bestehende Adresse
  • "Löschen" (Rot): Nur im Edit-Modus sichtbar, entfernt Adresse nach Bestätigung
  • "Abbrechen" (Grau): Schließt Dialog ohne Änderungen zu speichern

Visuelle Indikatoren

  • Dialog-Titel: "Neue Adresse" vs. "Adresse bearbeiten" zeigt aktuellen Modus
  • Vorausgefüllte Felder: Im Edit-Modus sind alle Daten bereits geladen
  • Focus-Indikator: Blaue Umrandung zeigt aktives Eingabefeld
  • Placeholder-Text: Beispiele in jedem Feld zur Orientierung

Häufige Probleme & Lösungen

Problem: Validierungsfehler bei korrekten Daten

Ursache: Unsichtbare Leerzeichen am Anfang oder Ende der Eingaben.

Lösung: Das System entfernt automatisch führende und nachfolgende Leerzeichen.

Problem: Dialog zeigt falsche Daten an

Ursache: Adresse wurde in anderem Modul zwischenzeitlich geändert.

Lösung: Dialog schließen und erneut öffnen für aktuelle Daten.

Problem: Adresse lässt sich nicht löschen

Ursache: Adresse wird noch in anderen Modulen verwendet (Foreign Key Constraint).

Lösung: Entfernen Sie zuerst alle Verknüpfungen, bevor Sie die Adresse löschen.

Best Practices

🏗️ Konsistente Adresseingabe

Verwenden Sie einheitliche Formate für bessere Datenqualität: "Straßenname Hausnummer" anstatt verschiedener Varianten.

🎯 Aussagekräftige Bezeichnungen

Nutzen Sie das Bezeichnungsfeld für eindeutige Namen, besonders bei mehreren Standorten in derselben Stadt.

⚡ Schnelle Navigation

Enter-Taste in jedem Eingabefeld speichert direkt - ideal für schnelle Erfassung mehrerer Adressen.

🔄 Command-System nutzen

Bei Fehlern können Sie über das Hauptmenü Undo verwenden, um Adress-Änderungen rückgängig zu machen.

Technische Details

Datenbank-Integration

Der Dialog arbeitet direkt mit dem Address-Schema und nutzt folgende Services:

  • address_commands: CRUD-Operationen über Command-Pattern
  • ContrExecUndoRedo: Undo/Redo-Controller für alle Operationen
  • db_services.Address: Direkte Datenbankzugriffe für Lese-Operationen

Schema-Kompatibilität

Unterstützte Datenstrukturen:

  • AddressCreate: Schema für neue Adress-Erstellung
  • Address: Vollständiges Schema für Updates
  • UUID-Handling: Eindeutige Identifikation über UUID-System

Signal-Architektur

Event-Kommunikation zwischen Komponenten:

  • address_saved(UUID): Signal bei erfolgreichem Speichern
  • address_deleted(UUID): Signal bei erfolgreichem Löschen
  • Automatische Propagation: Alle Module erhalten Benachrichtigungen
