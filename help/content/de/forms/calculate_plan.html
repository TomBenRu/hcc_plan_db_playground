Plan-Berechnung

Hilfe > Plan-Berechnung

Übersicht

Die automatische Plan-Berechnung erstellt optimierte Einsatzpläne basierend auf Verfügbarkeiten, Qualifikationen und Präferenzen. Das System nutzt fortschrittliche Optimierungsalgorithmen, um faire und effiziente Planungen zu generieren.

Hauptfunktionen

  • Automatische Optimierung - KI-basierte Planerstellung
  • Mehrere Vorschläge - Verschiedene Planungsalternativen
  • Faire Verteilung - Gleichmäßige Arbeitslasten
  • Konflikt-Erkennung - Automatische Problemidentifikation
  • Anpassbare Parameter - Berechnungszeiten konfigurieren

Grundbedienung

  1. Öffnen Sie die Plan-Berechnung über das Hauptmenü
  2. Das Fenster zeigt: "Schedule Creation [Team-Name]"
  3. Konfigurieren Sie die Parameter (siehe unten)
  4. Klicken Sie auf OK zum Start
  5. Überwachen Sie den Fortschritt im Progress-Dialog
  6. Wählen Sie die gewünschten Pläne aus den Ergebnissen aus

Tastaturkürzel

  • F1 - Diese Hilfe anzeigen
  • Enter - Berechnung starten
  • Escape - Dialog abbrechen
  • Tab - Zwischen Feldern wechseln
  • Ctrl+C - Berechnung abbrechen

Benutzeroberfläche

Parameter-Konfiguration

Period and Count (Zeitraum und Anzahl)

  • Planning Period - Wählen Sie den zu planenden Zeitraum:
    • Dropdown zeigt verfügbare Planungsperioden
    • Format: "DD.MM.YY - DD.MM.YY"
    • Nur Perioden mit Terminen sind verfügbar
  • Number of Planning Proposals - Anzahl der zu erstellenden Planvarianten:
    • Minimum: 1 Plan
    • Empfohlen: 3-5 Pläne für bessere Auswahl
    • Mehr Pläne = längere Berechnungszeit

Calculation Times (Berechnungszeiten in Sekunden)

Diese Parameter steuern, wie lange der Algorithmus für verschiedene Optimierungsschritte arbeitet:

  • Max. Time for Preprocessing - Vorberechnungen:
    • Standard: 20 Sekunden
    • Minimum: 5 Sekunden
    • Berechnet maximale Schichten pro Mitarbeiter
  • Max. Time for Fair Distribution Calculation - Faire Verteilung:
    • Automatisch berechnet: Anzahl Mitarbeiter × 50 Sekunden
    • Minimum: Anzahl Mitarbeiter × 5 Sekunden
    • Optimiert die Arbeitslasten-Verteilung
  • Max. Time for Plan Calculation - Hauptberechnung:
    • Standard: 60 Sekunden
    • Minimum: 5 Sekunden
    • Erstellt die finalen Planvorschläge

Berechnungsvorgang

Phasen der Berechnung

  1. Validierung - Prüfung der Eingabedaten
  2. Preprocessing - Vorberechnungen und Datenaufbereitung
  3. Fair Distribution - Berechnung fairer Arbeitszeiten
  4. Plan Generation - Erstellung der Planvorschläge
  5. Validation - Überprüfung auf Konflikte
  6. Saving - Speicherung der ausgewählten Pläne

Progress-Dialog

Während der Berechnung zeigt ein Fortschritts-Dialog:

  • Aktuelle Phase - Was gerade berechnet wird
  • Fortschrittsbalken - Geschätzter Gesamtfortschritt
  • Zeitanzeige - Verstrichene und geschätzte Restzeit
  • Cancel-Button - Berechnung abbrechen

Ergebnisse und Auswahl

Erfolgreiche Berechnung

Nach erfolgreicher Berechnung erhalten Sie:

  • Mehrere Planvorschläge - Je nach konfigurierter Anzahl
  • Auswahl-Dialog - Bestimmen Sie, wie viele Pläne gespeichert werden
  • Automatische Benennung - Pläne erhalten eindeutige Namen
  • Statistiken - Informationen über Arbeitslasten

Plan-Auswahl

  1. Dialog zeigt: "A total of X plans have been calculated"
  2. Wählen Sie die gewünschte Anzahl zu speichernder Pläne
  3. Standard: Alle berechneten Pläne speichern
  4. Bestätigen Sie mit OK
  5. Die Pläne werden automatisch gespeichert

Plan-Benennung

Pläne erhalten automatisch Namen im Format:

  • Format: "DD.MM.YY-DD.MM.YY (01)", "DD.MM.YY-DD.MM.YY (02)", etc.
  • Basis: Start- und Enddatum der Planungsperiode
  • Nummerierung: Aufsteigende Nummern für Eindeutigkeit
  • Konflikte: System verhindert Namenskonflikte automatisch

Praktische Anwendungsfälle

  • Notfallplanung - Schnelle Umplanung bei Personalausfällen oder Änderungen
  • Szenario-Vergleiche - Mehrere Planungsalternativen bewerten und optimale auswählen
  • Faire Verteilung - Gleichmäßige Arbeitsbelastung über längere Zeiträume sicherstellen

Häufige Probleme & Lösungen

⚠️ "Please select a period first"

Problem: Keine Planungsperiode ausgewählt
Lösung: Wählen Sie eine Planungsperiode aus dem Dropdown-Menü aus.

⚠️ "No plans can be created for the period"

Problem: Die gewählte Periode enthält keine Termine
Lösung: Erstellen Sie zuerst Termine in der Planungsansicht.

⚠️ "No solutions were found"

Ursachen und Lösungen:

  • Zu wenig Zeit: Erhöhen Sie die Berechnungszeiten
  • Unmögliche Anforderungen: Prüfen Sie Verfügbarkeiten und Qualifikationen
  • Konflikte: Überprüfen Sie Fixed Cast und Skill-Anforderungen

⚠️ "Fixed cast conflicts found"

Problem: Widersprüchliche Besetzungsanforderungen
Lösung: Korrigieren Sie die Fixed Cast-Einstellungen für die angezeigten Termine.

⚠️ "Skill conflicts found"

Problem: Nicht genügend Mitarbeiter mit erforderlichen Qualifikationen
Lösung: Erweitern Sie das Team oder reduzieren Sie die Skill-Anforderungen.

⚠️ Berechnung abbrechen

Wichtig: Ein Abbruch verwirft alle bisherigen Ergebnisse. Lassen Sie komplexe Berechnungen nach Möglichkeit vollständig durchlaufen.

Best Practices

💡 Zeitoptimierung

Längere Berechnungszeiten führen meist zu besseren Ergebnissen. Für komplexe Planungen empfehlen wir mindestens 120 Sekunden für die Hauptberechnung.

🎯 Bessere Ergebnisse erzielen

  • Vollständige Stammdaten: Alle Mitarbeiter und Qualifikationen pflegen
  • Realistische Verfügbarkeiten: Korrekte Arbeitszeiten definieren
  • Ausreichend Personal: Mehr Mitarbeiter als Mindestbesetzung
  • Flexible Anforderungen: Nicht zu strenge Fixed Cast-Regeln
  • Längere Berechnungszeiten: Besonders bei komplexen Planungen

🚀 Performance-Tipps

  • Kleinere Perioden: Teilen Sie große Zeiträume auf
  • Weniger Varianten: Beginnen Sie mit 2-3 Planvorschlägen
  • Schrittweise Optimierung: Erst Grundplan, dann Verfeinerung
  • Zwischenspeichern: Speichern Sie gute Teilergebnisse
