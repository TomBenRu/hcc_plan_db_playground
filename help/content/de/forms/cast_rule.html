Cast-Regel bearbeiten

Hilfe > Cast-Regel bearbeiten

Übersicht

Der Cast-Regel-Dialog ermöglicht das Erstellen und Bearbeiten von wiederverwendbaren Besetzungsregeln. Mit Cast-Regeln können Sie definieren, wie sich die Besetzung über eine Serie von Terminen hinweg verhalten soll - von identischer Wiederholung bis hin zu bewusster Variation.

Hauptfunktionen

  • Regel-Definition - Erstellung präziser Cast-Patterns mit Symbolen
  • Automatische Vereinfachung - System optimiert Regeln für bessere Performance
  • Wiederverwendbarkeit - Regeln können in mehreren Cast-Gruppen verwendet werden
  • Validierung - Echtzeit-Prüfung auf Duplikate und Konflikte
  • Pattern-Erkennung - Automatische Erkennung von sich wiederholenden Mustern

Grundbedienung

Neue Cast-Regel erstellen

  1. Geben Sie einen aussagekräftigen Namen für die Regel ein
  2. Definieren Sie das Cast-Pattern mit den Symbolen *, ~, -
  3. System konvertiert automatisch zu Großbuchstaben
  4. Validierung prüft auf Duplikate und Konflikte
  5. Speichern Sie die Regel für Wiederverwendung

Cast-Regel bearbeiten

  1. Name und Pattern sind vorausgefüllt mit aktuellen Werten
  2. Nehmen Sie gewünschte Änderungen vor
  3. System prüft auf Konflikte mit anderen Regeln
  4. Speichern übernimmt die Änderungen für alle verwendenden Cast-Gruppen

Tastaturkürzel

  • F1 - Diese Hilfe öffnen
  • Tab - Zwischen Name und Cast-Regel wechseln
  • Strg+A - Gesamten Text im aktiven Feld markieren
  • Strg+S - Speichern
  • Escape - Dialog abbrechen

Verwandte Funktionen

  • Cast-Regeln Management - Übersicht und Verwaltung aller Regeln
  • Cast-Gruppen-Eigenschaften - Anwendung der Regeln auf Gruppen
  • Fixed Cast Builder - Alternative für feste Besetzungsdefinitionen

Anwendungsbereich

Cast-Regeln werden verwendet für:

  • Besetzungsmuster - Definition wiederkehrender Cast-Patterns über mehrere Termine
  • Variation Control - Steuerung von Besetzungsabwechslung und -konsistenz
  • Automatisierte Planung - Regelbasierte Cast-Zuweisung für Event-Serien
  • Template-System - Wiederverwendbare Patterns für verschiedene Projekte

Benutzeroberfläche

Name-Feld

  • Eindeutigkeit - Name muss projektweise eindeutig sein
  • Beschreibend - Verwenden Sie aussagekräftige Namen
  • Beispiele - "Konstant", "Wechselnd", "Haupt-Ersatz", "Rotierend"

Cast-Regel-Feld

  • Symbole - Nur *, ~, - sind erlaubt
  • Großschreibung - System konvertiert automatisch
  • Vereinfachung - Redundante Patterns werden automatisch optimiert
  • Validierung - Echtzeit-Prüfung der Eingabe

Cast-Regel-Symbole

Grundlegende Symbole

  • * (Asterisk) - Beliebige Besetzung (keine Einschränkungen)
  • ~ (Tilde) - Gleiche Besetzung wie beim vorherigen Termin
  • - (Minus) - Andere Besetzung als beim vorherigen Termin

Symbol-Bedeutung im Detail

  • * - "Any Cast" (System wählt optimal verfügbare Akteure)
  • ~ - "Same Cast" (Exakt dieselben Personen wie zuvor)
  • - - "Different Cast" (Bewusst andere Personen als zuvor)

Praktische Anwendungsfälle

Standard-Regel für Projekt erstellen

  1. Öffnen Sie den Dialog über Neue Cast-Regel
  2. Name: Standard Rotation
  3. Pattern: ~- (abwechselnd gleich/anders)
  4. Speichern Sie die Regel
  5. Regel steht allen Cast-Gruppen zur Verfügung

Spezial-Regel für Event-Serie

  1. Analysieren Sie die gewünschte Besetzungs-Logik
  2. Übersetzen Sie in Symbol-Pattern
  3. Testen Sie das Pattern gedanklich mit mehreren Terminen
  4. Erstellen Sie die Regel mit beschreibendem Namen
  5. Wenden Sie die Regel auf entsprechende Cast-Gruppen an

Häufige Probleme & Lösungen

⚠️ Name bereits vorhanden

Problem: Regelname ist bereits vergeben
Ursache: Jeder Regel-Name muss projektweise eindeutig sein
Lösung: Verwenden Sie beschreibende Zusätze (z.B. "Rotation_v2") oder prüfen Sie bestehende Regel-Namen im Management-Dialog

⚠️ Regel bereits vorhanden

Problem: Identisches Pattern existiert bereits
Ursache: System erkennt Duplikat-Patterns automatisch
Lösung: Verwenden Sie die bestehende Regel oder definieren Sie ein anderes Pattern. System zeigt den Namen der existierenden Regel an

⚠️ Regel wird zu "None" vereinfacht

Problem: Pattern vereinfacht sich zu einem einfachen "*"
Bedeutung: "Keine spezielle Regel" - beliebige Besetzung
Lösung: Definieren Sie ein spezifischeres Pattern für gewünschte Cast-Kontrolle

Best Practices

🎯 Regel-Design

  • Einfachheit: Beginnen Sie mit einfachen Patterns
  • Vorhersagbarkeit: Zu komplexe Patterns können verwirren
  • Dokumentation: Verwenden Sie selbsterklärende Namen

🚀 Performance-Überlegungen

  • Kurze Patterns: System muss weniger wiederholen
  • Eindeutige Regeln: Vermeiden Sie übermäßig ähnliche Patterns
  • Standard-Regeln: Erstellen Sie wiederverwendbare Basis-Patterns

💡 Pattern-Beispiele

Einfache Patterns:

  • "~" - Immer dieselbe Besetzung für alle Termine
  • "-" - Immer andere Besetzung (maximale Variation)
  • "*" - Beliebige Besetzung (System entscheidet optimal)

Erweiterte Patterns:

  • "~-" - Gleich, dann anders, dann wieder gleich, dann anders...
  • "~~-" - Zweimal gleich, dann anders, dann wieder zweimal gleich...
  • "~--" - Einmal gleich, dann zweimal anders, dann wiederholen
  • "*~-" - Beliebig, dann gleich, dann anders, dann wiederholen

⚡ Praktische Anwendungsfälle

  • "Konstante Besetzung": "~" für durchgehend gleichen Cast
  • "Wechselnde Teams": "~-" für abwechselnde Besetzung
  • "Haupt + Ersatz": "~~-" für hauptsächlich gleichen Cast mit gelegentlicher Variation
  • "Flexible Planung": "*" für System-optimierte Zuteilung

Erweiterte Funktionen

Automatische Vereinfachung

Pattern-Optimierung

  • "~~~" → "~" - Wiederholungen werden erkannt und vereinfacht
  • "*~*~" → "*~" - Sich wiederholende Subsequenzen werden reduziert
  • "***" → "*" - Redundante Symbole werden entfernt

Null-Vereinfachung vermeiden

  • "*" allein → None - Wird als "keine spezielle Regel" interpretiert
  • Leere Patterns - Werden als ungültig erkannt
  • System-Warnung - Bei problematischen Vereinfachungen

Erweiterte Pattern-Logik

Wiederholungs-Erkennung

  • Automatische Erkennung - System erkennt sich wiederholende Subsequenzen
  • Optimierung - Lange Patterns werden auf kürzeste Form reduziert
  • Beispiel - "~-~-~-" wird zu "~-" vereinfacht

Komplexe Szenarien

  • Asymmetrische Rotation - "~~-" für 2:1 Verhältnis
  • Gelegentliche Variation - "~~~~-" für seltene Änderungen
  • Strukturierte Abwechslung - "~-~*" für gemischte Strategien

Integration mit Cast-System

Verwendung in Cast-Gruppen

  • Regel-Auswahl - Gespeicherte Regeln erscheinen in Dropdown-Listen
  • Überschreibung - Custom-Regeln können temporär überschreiben
  • Vererbung - Child-Gruppen können Parent-Regeln übernehmen

Regel-Priorität

  • 1. Custom Rule - Direkt in Cast-Gruppe definiert
  • 2. Cast Rule - Aus diesem Dialog stammende Regel
  • 3. Parent Rule - Von übergeordneter Gruppe geerbt

Technische Details

System-Architektur

  • Pattern-Engine - Automatische Erkennung wiederholender Subsequenzen
  • Validation-System - Echtzeit-Prüfung auf Duplikate und Konsistenz
  • Command-Pattern - Undo-fähige Operationen
  • String-Processing - Automatische Groß-/Kleinschreibung-Normalisierung

Datenmodell

  • Rule Repository - Zentrale Speicherung aller Cast-Regeln
  • Reference Counting - Verfolgung der Regel-Verwendung in Cast-Gruppen
  • Pattern Validation - Syntax-Prüfung und Optimierung

Performance-Optimierung

  • Pattern Caching - Häufig verwendete Patterns werden gecacht
  • Lazy Evaluation - Patterns werden nur bei Bedarf ausgewertet
  • Memory Efficiency - Optimierte Speicherung wiederkehrender Muster

Support

  • Nutzen Sie aussagekräftige Namen für bessere Übersichtlichkeit
  • Testen Sie Patterns gedanklich mit mehreren Terminen
  • Konsultieren Sie das Cast-Regeln Management für Übersicht aller Regeln
