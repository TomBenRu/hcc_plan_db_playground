Cast-Regeln Verwaltung

Hilfe > Cast-Regeln Verwaltung

Übersicht

Die Cast-Regeln-Verwaltung bietet eine zentrale Übersicht über alle projektspezifischen Besetzungsregeln. Sie können neue Regeln erstellen, bestehende bearbeiten und nicht mehr benötigte Regeln löschen. Alle Regeln stehen automatisch in Cast-Gruppen zur Verfügung.

Hauptfunktionen

  • Regel-Übersicht - Tabelle aller verfügbaren Cast-Regeln mit Name und Pattern
  • Regel-Erstellung - Schneller Zugang zum Erstellen neuer Regeln
  • Regel-Bearbeitung - Modifikation bestehender Regel-Definitionen
  • Regel-Löschung - Entfernen nicht mehr benötigter Regeln
  • Automatische Sortierung - Alphabetische Anordnung nach Regel-Namen
  • Referenz-Prüfung - Schutz vor Löschung verwendeter Regeln

Grundbedienung

Neue Cast-Regel erstellen

  1. Klicken Sie auf Neu...
  2. Der Cast-Regel-Editor öffnet sich
  3. Definieren Sie Name und Pattern
  4. Bestätigen Sie die Erstellung
  5. Neue Regel erscheint automatisch in der Tabelle

Bestehende Regel bearbeiten

  1. Auswahl - Klicken Sie auf eine Zeile in der Tabelle
  2. Bearbeitung - Klicken Sie Bearbeiten... oder Doppelklick
  3. Nehmen Sie Änderungen vor
  4. Bestätigen Sie die Aktualisierung
  5. Änderungen werden in allen verwendenden Cast-Gruppen übernommen

Tastaturkürzel

  • F1 - Diese Hilfe öffnen
  • Enter - Ausgewählte Regel bearbeiten
  • Delete - Ausgewählte Regel löschen (mit Bestätigung)
  • Strg+N - Neue Regel erstellen
  • F2 - Regel bearbeiten (wenn ausgewählt)
  • Escape - Dialog schließen

Verwandte Funktionen

  • Cast-Regel bearbeiten - Erstellen und Bearbeiten einzelner Regeln
  • Cast-Gruppen-Eigenschaften - Anwendung der Regeln auf Cast-Gruppen
  • Cast-Gruppen - Verwaltung der Gruppen die Regeln verwenden

Anwendungsbereich

Die Cast-Regeln-Verwaltung wird verwendet für:

  • Zentrales Regel-Management - Übersicht und Verwaltung aller projektspezifischen Besetzungsregeln
  • Regel-Bibliothek - Aufbau wiederverwendbarer Pattern-Templates
  • Konsistenz-Management - Einheitliche Cast-Patterns projekt-weit
  • Referenz-Kontrolle - Überwachung der Regel-Verwendung in Cast-Gruppen

Benutzeroberfläche

Regel-Tabelle

  • Name-Spalte - Anzeige aller Regel-Namen (alphabetisch sortiert)
  • Pattern-Spalte - Darstellung der Cast-Regel-Symbole
  • Zeilen-Auswahl - Klick auf Zeile aktiviert Bearbeitung/Löschung
  • Doppelklick - Direkter Zugang zur Regel-Bearbeitung
  • Auto-Resize - Spalten passen sich automatisch an Inhalt an

Aktions-Buttons

  • Neu... - Öffnet Dialog zum Erstellen einer neuen Cast-Regel
  • Bearbeiten... - Öffnet Editor für ausgewählte Regel (Zeile muss ausgewählt sein)
  • Löschen... - Entfernt ausgewählte Regel (mit Referenz-Prüfung)
  • OK - Schließt Dialog und übernimmt alle Änderungen
  • Abbrechen - Verwirft alle Änderungen (Undo-System)

Praktische Anwendungsfälle

Projekt-weite Regel-Standardisierung

  1. Analysieren Sie häufig verwendete Cast-Patterns
  2. Erstellen Sie Standard-Regeln für jedes Pattern:
  3. "Konstant" → "~"
  4. "Abwechselnd" → "~-"
  5. "Hauptsächlich gleich" → "~~~-"
  6. "Flexibel" → "*"
  7. Dokumentieren Sie die Regeln im Team
  8. Verwenden Sie konsistent in allen Cast-Gruppen

Regel-Bereinigung und Optimierung

  1. Überprüfen Sie regelmäßig die Regel-Liste
  2. Identifizieren Sie nicht verwendete Regeln
  3. Prüfen Sie auf redundante oder sehr ähnliche Patterns
  4. Konsolidieren Sie ähnliche Regeln
  5. Löschen Sie überflüssige Definitionen

Häufige Probleme & Lösungen

⚠️ Regel kann nicht gelöscht werden

Problem: Löschen-Button reagiert nicht oder zeigt Fehlermeldung
Ursache: Regel wird noch in Cast-Gruppen verwendet (Referenz-Check)
Lösung: Entfernen Sie die Regel zuerst aus allen Cast-Gruppen oder ersetzen Sie sie durch eine andere

⚠️ Tabelle lädt nicht richtig

Problem: Regel-Tabelle zeigt keine oder veraltete Daten
Ursachen:

  • Parallel laufende Änderungen in anderen Fenstern
  • Datenbank-Synchronisations-Problem
  • Cache-Problem in der Anwendung

Lösung: Schließen und öffnen Sie den Dialog neu oder starten Sie die Anwendung neu

⚠️ Bearbeitung funktioniert nicht

Problem: Bearbeiten... Button ist deaktiviert
Ursachen:

  • Keine Tabellenzeile ausgewählt
  • Fehlende Schreibrechte
  • Regel ist bereits gelöscht

Lösung: Klicken Sie auf eine Tabellenzeile und prüfen Sie Ihre Berechtigungen

Best Practices

🎯 Namens-Konventionen

  • Beschreibend: "Wechselnd_2zu1" statt "Pattern1"
  • Kategorien: Präfixe für verschiedene Regel-Typen
  • Versionierung: "_v2" für überarbeitete Regeln

🚀 Regel-Bibliothek aufbauen

  • Standard-Set: Basis-Regeln für häufige Szenarien
  • Spezial-Regeln: Event-spezifische Patterns
  • Template-Regeln: Ausgangspunkt für Anpassungen

💡 Regel-Management-Tipps

  • Regelmäßige Bereinigung: Entfernen Sie ungenutzte Regeln
  • Konsistente Verwendung: Nutzen Sie Standard-Regeln projekt-weit
  • Dokumentation: Verwenden Sie aussagekräftige Regel-Namen
  • Versionierung: Behalten Sie bewährte Regeln als Backup

Erweiterte Funktionen

Regel-Referenzen und Abhängigkeiten

Verwendungs-Tracking

  • Cast-Gruppen - Zeigt, welche Gruppen die Regel verwenden
  • Lösch-Schutz - Verwendete Regeln können nicht gelöscht werden
  • Update-Propagation - Regel-Änderungen wirken sich sofort aus

Abhängigkeits-Management

  • Vor Löschung - System prüft alle Referenzen
  • Warnung - Information über verwendende Cast-Gruppen
  • Alternative - Regel erst aus Cast-Gruppen entfernen, dann löschen

Regel-Organisation

Sortierung und Filterung

  • Alphabetische Sortierung - Automatisch nach Regel-Namen
  • Pattern-Gruppierung - Ähnliche Patterns werden zusammengefasst
  • Verwendungs-Status - Unterscheidung zwischen genutzten und ungenutzten Regeln

Bulk-Operationen

  • Mehrfach-Auswahl - Auswahl mehrerer Regeln für Batch-Operationen
  • Massenbearbeitung - Änderungen an mehreren Regeln gleichzeitig
  • Export/Import - Regel-Sets zwischen Projekten übertragen

Integration in Arbeitsabläufe

Regel-Templates

  • Standard-Templates - Vordefinierte Regel-Sets für häufige Szenarien
  • Projekt-spezifische Templates - Angepasste Regel-Sammlungen
  • Template-Import - Übernahme bewährter Regel-Sets

Qualitätssicherung

  • Regel-Validierung - Automatische Prüfung auf Pattern-Konsistenz
  • Duplikats-Erkennung - Warnung bei identischen Patterns
  • Best-Practice-Hints - Vorschläge für Regel-Optimierungen

Technische Details

System-Architektur

  • Datenbank-Integration - Projektspezifische Speicherung aller Regeln
  • Referenz-Integrität - Automatische Prüfung auf Regel-Verwendung
  • Soft-Delete - Regeln werden zunächst zum Löschen markiert
  • Undo-System - Alle Management-Operationen sind rückgängig machbar

Performance-Optimierung

  • Lazy Loading - Regel-Details werden nur bei Bedarf geladen
  • Caching - Häufig verwendete Regeln werden im Speicher gehalten
  • Indexierung - Optimierte Datenbankabfragen für große Regel-Sets

Datenmodell

  • Rule Repository - Zentrale Verwaltung aller Cast-Regeln
  • Reference Tracking - Verfolgung der Verwendung in Cast-Gruppen
  • Version Control - Änderungshistorie für Regel-Entwicklung

Support

Bei Problemen mit der Cast-Regeln-Verwaltung:

  • Organisieren Sie eine gut strukturierte Regel-Bibliothek für bessere Übersichtlichkeit
  • Nutzen Sie aussagekräftige Namen für einfacheres Wiederfinden
  • Prüfen Sie regelmäßig auf ungenutzte Regeln zur System-Optimierung
  • Konsultieren Sie die Cast-Regel Bearbeitung für Details zur Pattern-Erstellung
