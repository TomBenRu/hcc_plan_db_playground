Standort-Kombinationen bearbeiten

Hilfe > Standort-Kombinationen bearbeiten

Übersicht

Dieser Dialog ermöglicht die Verwaltung und Bearbeitung bestehender Standort-Kombinationen für Teams, Personen oder Planungsperioden. Im Gegensatz zum Erstellungs-Dialog bietet diese Ansicht ein komplettes Management-Interface mit datum-abhängiger Anzeige, Parent-Model-Vererbung und erweiterten Bearbeitungsfunktionen.

Hauptfunktionen

  • Datum-basierte Anzeige: Auswahl eines Datums bestimmt, welche Location-Kombinationen verfügbar und relevant sind
  • Tabellen-Management: Übersichtliche Liste aller aktuellen Kombinationen mit Details zu Standorten und Reisezeiten
  • Parent-Model-Vererbung: Automatische Übernahme von Kombinationen aus übergeordneten Modellen (z.B. Team → Person)
  • Neue Kombinationen erstellen: Direkter Zugang zum Erstellungs-Dialog für neue Location-Kombinationen
  • Reset-Funktion: Zurücksetzen auf die Kombinationen des Parent-Models
  • Selektives Löschen: Entfernung einzelner Kombinationen ohne Auswirkung auf Parent-Models
  • Intelligente Info-Anzeige: Automatische Benachrichtigungen über Team-Zuordnungen und Facility-Verfügbarkeit

Grundbedienung

  1. Dialog öffnen: Zugang über die entsprechende Funktion in Team-, Personen- oder Planungsperioden-Management
  2. Datum auswählen: Setzen Sie das gewünschte Datum im "Datum"-Feld - dies bestimmt die verfügbaren Teams und Locations
  3. Kombinationen anzeigen: Die Tabelle wird automatisch mit den für das gewählte Datum relevanten Kombinationen gefüllt
  4. Neue Kombination hinzufügen:
    • Klicken Sie "New..." um den Erstellungs-Dialog zu öffnen
    • Konfigurieren Sie die gewünschten Standorte und Reisezeiten
    • Die neue Kombination erscheint automatisch in der Tabelle
  5. Kombinationen bearbeiten/löschen:
    • Wählen Sie eine Zeile in der Tabelle aus
    • Klicken Sie "Delete" um die Kombination zu entfernen
    • Oder nutzen Sie "Reset" um alle auf Parent-Model zurückzusetzen
  6. Änderungen speichern: Klicken Sie "Save" zum Übernehmen oder "Cancel" zum Verwerfen

Tastaturkürzel

  • F1: Diese Hilfe öffnen
  • Tab: Zwischen Eingabefeldern und Buttons navigieren
  • Enter: Dialog bestätigen (entspricht "Save")
  • Escape: Dialog abbrechen
  • Pfeiltasten: In der Tabelle navigieren
  • Delete: Ausgewählte Kombination löschen (bei fokussierter Tabelle)
  • Ctrl+N: Neue Kombination erstellen (entspricht "New...")
  • Ctrl+R: Reset auf Parent-Model (entspricht "Reset")

Verwandte Funktionen

  • Neue Standort-Kombination - Erstellung neuer Location-Kombinationen
  • Team-Verwaltung - Konfiguration von Teams und deren Zuordnungen
  • Termine bearbeiten - Verwendung der Location-Kombinationen in der Terminplanung

Anwendungsbereich

Die Standort-Kombinationen-Verwaltung ist zentral für die Organisation von Teams, die an mehreren Arbeitsplätzen tätig sind. Sie ermöglicht eine hierarchische Struktur der Location-Zuordnungen und berücksichtigt zeitliche Veränderungen in der Team-Zusammensetzung.

Typische Verwendungsszenarien:

  • Team-Level-Management: Definition von Standard-Kombinationen für ganze Teams
  • Person-spezifische Anpassungen: Individuelle Überschreibung von Team-Kombinationen für einzelne Mitarbeiter
  • Planungsperioden-Konfiguration: Zeitraum-spezifische Location-Kombinationen für Projekte
  • Verfügbarkeits-Management: Anpassung an sich ändernde Team-Zuordnungen über die Zeit

Benutzeroberfläche

Datum-Steuerung

Der obere Bereich enthält ein Datum-Auswahlfeld, das eine Schlüsselrolle spielt:

  • Funktion: Bestimmt welche Teams dem aktuellen Model zugeordnet sind
  • Auswirkungen: Ändert die verfügbaren Locations und die angezeigten Kombinationen
  • Minimum-Datum: Auf heutiges Datum oder später beschränkt für zukunftsorientierte Planung
  • Automatische Updates: Bei Datum-Änderung wird die komplette Anzeige neu geladen

Info-Bereich

Oberhalb der Tabelle zeigt ein Info-Label wichtige Statusmeldungen:

  • "Kein Team zugeordnet": Warnung wenn keine Team-Zuordnung für das gewählte Datum existiert
  • "Gleiche Facilities": Bestätigung dass das Team über alle Planungsperioden-Tage konsistente Standorte hat
  • "Unterschiedliche Facilities": Warnung bei variierenden Standort-Zuordnungen über die Zeit

Kombinationen-Tabelle

Das Hauptelement ist eine sortierbare Tabelle mit drei Spalten:

  • ID-Spalte: Versteckte eindeutige Identifikation (für interne Referenzen)
  • Facility Combination: Anzeige der kombinierten Standorte im Format "Standort A (Stadt) + Standort B (Stadt)"
  • Time between assignments: Konfigurierte Reisezeit zwischen den Standorten

Management-Buttons

Unterhalb der Tabelle befinden sich die Aktions-Buttons:

  • "New...": Öffnet den Dialog zur Erstellung neuer Kombinationen
  • "Reset": Stellt den ursprünglichen Zustand basierend auf Parent-Model wieder her
  • "Delete": Entfernt die aktuell ausgewählte Kombination aus der Liste

Praktische Anwendungsfälle

🏢 Team-basierte Standard-Konfiguration

Szenario: Ein Service-Team hat Standard-Kombinationen, aber einzelne Mitarbeiter brauchen Anpassungen

Vorgehen:

  1. Öffnen Sie die Standort-Kombinationen-Verwaltung auf Person-Ebene
  2. Wählen Sie ein relevantes Datum für die Team-Zuordnung
  3. Prüfen Sie die automatisch geerbten Team-Kombinationen
  4. Fügen Sie mit "New..." person-spezifische Kombinationen hinzu
  5. Löschen Sie irrelevante Team-Kombinationen mit "Delete"

🚀 Projektphasen-Management

Szenario: Ein Projekt hat verschiedene Phasen mit unterschiedlichen Location-Anforderungen

Vorgehen:

  1. Öffnen Sie die Verwaltung auf ActorPlanPeriod-Ebene
  2. Wählen Sie ein Datum innerhalb der gewünschten Projektphase
  3. Nutzen Sie "Reset" um von Standard-Kombinationen auszugehen
  4. Passen Sie mit "New..." die Kombinationen an die Projektphase an
  5. Wiederholen Sie für verschiedene Phasen mit unterschiedlichen Daten

🔄 Team-Wechsel-Szenarien

Szenario: Ein Mitarbeiter wechselt zwischen verschiedenen Teams und benötigt datum-spezifische Kombinationen

Vorgehen:

  1. Wählen Sie das Datum vor dem Team-Wechsel
  2. Konfigurieren Sie die Kombinationen für das erste Team
  3. Wählen Sie das Datum nach dem Team-Wechsel
  4. Beobachten Sie die automatische Anpassung der verfügbaren Locations
  5. Passen Sie die Kombinationen entsprechend dem neuen Team an

🎯 Union-Logic für Planungsperioden

Szenario: Ein ActorPlanPeriod umfasst mehrere Tage mit variierenden Team-Zuordnungen

Vorgehen:

  1. Öffnen Sie die Verwaltung für die entsprechende Planungsperiode
  2. Das System bildet automatisch die Union aller verfügbaren Locations über alle Tage
  3. Beachten Sie die Info-Meldung über konsistente/variable Facility-Zuordnungen
  4. Erstellen Sie Kombinationen basierend auf der erweiterten Location-Menge
  5. Berücksichtigen Sie dabei die längsten möglichen Reisezeiten

Häufige Probleme & Lösungen

Problem: "Tabelle bleibt leer nach Datum-Auswahl"

Ursache: Kein Team ist der Person/dem Model für das gewählte Datum zugeordnet

Lösung: Überprüfen Sie die Team-Zuordnungen für das gewählte Datum oder wählen Sie ein anderes Datum mit bekannter Team-Zuordnung

Problem: "Reset-Button ohne Wirkung"

Ursache: Kein Parent-Model verfügbar (z.B. bei Team-Level-Management)

Lösung: Reset funktioniert nur bei abgeleiteten Models (Person von Team, ActorPlanPeriod von Person). Bei Top-Level-Models ist Reset nicht verfügbar

Problem: "Kombinationen verschwinden nach Datum-Änderung"

Ursache: Die Standorte der Kombinationen sind für das neue Datum nicht verfügbar

Lösung: Kombinationen werden nur angezeigt, wenn alle beteiligten Standorte dem Team am gewählten Datum zugeordnet sind

Problem: "Neue Kombinationen können nicht erstellt werden"

Ursache: Weniger als zwei Facilities sind dem Team für das gewählte Datum zugeordnet

Lösung: Überprüfen Sie die Team-Location-Zuordnungen oder wählen Sie ein anderes Datum mit ausreichenden Facility-Zuordnungen

Problem: "Delete funktioniert nicht"

Ursache: Keine Zeile in der Tabelle ausgewählt

Lösung: Klicken Sie zunächst auf eine Zeile in der Tabelle um sie auszuwählen, dann auf "Delete"

Best Practices

💡 Datum-Management-Strategien

  • Referenz-Daten verwenden: Nutzen Sie bekannte Daten mit stabilen Team-Zuordnungen als Ausgangspunkt
  • Planungsperioden-Start: Bei ActorPlanPeriod-Management verwenden Sie den Startag der Periode
  • Team-Wechsel-Daten: Dokumentieren Sie wichtige Team-Wechsel-Termine für bessere Datum-Auswahl
  • Zukunftsorientierung: Konzentrieren Sie sich auf zukünftige Daten für aktive Planungen

🚀 Parent-Model-Vererbung optimal nutzen

  • Hierarchische Planung: Definieren Sie Standard-Kombinationen auf Team-Level, spezifische auf Person-Level
  • Reset als Baseline: Verwenden Sie Reset um zu bewährten Konfigurationen zurückzukehren
  • Schrittweise Anpassung: Beginnen Sie mit Reset, dann selektive Ergänzungen/Löschungen
  • Konsistenz bewahren: Dokumentieren Sie Abweichungen von Team-Standards

🏗️ Effizientes Kombinationen-Management

  • Tabellen-Navigation: Nutzen Sie die Sortier-Funktion der Tabelle für bessere Übersicht
  • Batch-Operations: Gruppieren Sie ähnliche Änderungen für effiziente Bearbeitung
  • Validation vor Speichern: Überprüfen Sie alle Kombinationen in der Tabelle vor dem Speichern
  • Backup-Strategie: Notieren Sie wichtige Konfigurationen bevor Sie größere Änderungen vornehmen

🔄 Union-Logic bei Planungsperioden

  • Info-Meldungen beachten: Achten Sie auf Warnungen über variierende Facility-Zuordnungen
  • Konservative Reisezeiten: Bei Union-Scenarios wählen Sie längere Reisezeiten für Sicherheit
  • Tages-weise Validierung: Überprüfen Sie bei kritischen Perioden einzelne Tage
  • Flexibilität planen: Berücksichtigen Sie mögliche Team-Änderungen während der Periode

Erweiterte Funktionen

Parent-Model-Factory Pattern

Der Dialog nutzt ein ausgeklügeltes Factory-Pattern für verschiedene Anwendungsszenarien:

  • Team-Management: Keine Parent-Factory - Team ist Top-Level
  • Person-Management: Team als Parent-Model für Vererbung
  • ActorPlanPeriod-Management: Person als Parent-Model mit Planungsperioden-Logik
  • AvailDay-Management: Verschiedene Parent-Models je nach Kontext

Dynamische Team-Zuordnung

Das System berücksichtigt zeitlich variable Team-Zuordnungen:

  • team_at_date_factory: Ermittlung des relevanten Teams für ein spezifisches Datum
  • Automatische Anpassung: Location-Sets passen sich an Team-Änderungen an
  • Konsistenz-Prüfung: Warnung bei inkonsistenten Team-Location-Zuordnungen

Command Pattern Integration

Alle Änderungen nutzen das Command Pattern für Undo/Redo-Funktionalität:

  • Transactional Safety: Alle Änderungen können rückgängig gemacht werden
  • Batch Commands: Mehrere Operationen werden als Einheit behandelt
  • Model-spezifische Commands: Verschiedene Command-Typen für verschiedene Model-Typen

Technische Details

ModelWithCombLocPossible Integration

Der Dialog arbeitet mit verschiedenen Models, die das Interface ModelWithCombLocPossible implementieren:

  • TeamShow: Team-Level-Kombinationen als Basis
  • PersonShow: Person-spezifische Kombinationen mit Team-Vererbung
  • ActorPlanPeriodShow: Planungsperioden-spezifische Kombinationen mit Union-Logic
  • AvailDay/AvailDayShow: Verfügbarkeits-basierte Kombinationen

Factory Command Pattern

Dynamische Command-Erstellung basierend auf Model-Type:

  • factory_for_put_in_combs: Model-spezifische "Put-In"-Commands
  • factory_for_remove_combs: Model-spezifische "Remove"-Commands
  • Error Handling: KeyError-Behandlung für unbekannte Model-Types
  • Erweiterbarkeit: Einfache Ergänzung neuer Model-Types

Union-Logic Implementation

Spezielle Logik für ActorPlanPeriod-Management:

  • Tages-Union: Vereinigung aller Location-Sets über alle Planungsperioden-Tage
  • Team-Konsistenz-Prüfung: Validierung ob Team über die gesamte Periode konstant ist
  • Location-ID-Sammlung: Set-basierte Sammlung für Eindeutigkeit
  • Info-Feedback: User-Information über Konsistenz-Status

Datum-abhängige Location-Ermittlung

Intelligente Location-Ermittlung basierend auf Datum und Team-Zuordnungen:

  • get_locations_of_team_at_date: Datum-spezifische Team-Location-Zuordnungen
  • get_curr_assignment_of_person: Person-Team-Zuordnung für spezifisches Datum
  • Caching-Mechanismen: Optimierte Datenbankabfragen
  • Filter-Logic: Ausschluss nicht verfügbarer oder gelöschter Locations
