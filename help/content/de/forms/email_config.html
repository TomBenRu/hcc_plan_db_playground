Email Configuration Dialog

Hilfe > E-Mail-System > Email Configuration

Übersicht

Der Email Configuration Dialog ermöglicht die vollständige Einrichtung der SMTP-E-Mail-Server-Verbindung. Hier konfigurieren Sie alle notwendigen Parameter für den E-Mail-Versand aus HCC Plan.

Hauptfunktionen

  • SMTP-Server-Einrichtung - Vollständige Server-Konfiguration
  • Authentifizierung - Benutzername und Passwort-Verwaltung
  • Verschlüsselung - TLS/SSL/STARTTLS-Einstellungen
  • Absender-Konfiguration - Name und E-Mail-Adresse
  • Test-Funktionen - Verbindung testen vor dem Speichern
  • Sichere Speicherung - Verschlüsselte Passwort-Speicherung

Grundbedienung

  1. Öffnen Sie "Emails" → "Email Configuration..." aus dem Hauptmenü
  2. Geben Sie den SMTP-Server ein (z.B. smtp.gmail.com)
  3. Setzen Sie den Port (standardmäßig 587 für STARTTLS)
  4. Konfigurieren Sie die Absender-Daten (Name und E-Mail)
  5. Geben Sie Benutzername und Passwort ein
  6. Wählen Sie die Verschlüsselung (meist STARTTLS)
  7. Klicken Sie auf "Verbindung testen" zur Überprüfung
  8. Bei erfolgreichem Test: "Speichern" klicken

Tastaturkürzel

F1 - Diese Hilfe öffnen

Tab - Zwischen Eingabefeldern wechseln

Enter - Konfiguration speichern

Escape - Dialog abbrechen

  • E-Mail-System Übersicht - Alle E-Mail-Funktionen im Überblick

Konfigurationsfelder

Server-Einstellungen

  • SMTP-Server - Hostname oder IP-Adresse des E-Mail-Servers
  • Port - SMTP-Port (587 für STARTTLS, 465 für SSL, 25 für unverschlüsselt)
  • Verschlüsselung - Sicherheitsprotokoll (STARTTLS empfohlen)

Absender-Informationen

  • Absender-Name - Anzeigename für ausgehende E-Mails
  • Absender-E-Mail - E-Mail-Adresse des Absenders

Authentifizierung

  • Benutzername - SMTP-Authentifizierung Benutzername
  • Passwort - SMTP-Authentifizierung Passwort

Häufige SMTP-Einstellungen

📧 Gmail

Server: smtp.gmail.com
Port: 587
Verschlüsselung: STARTTLS
Hinweis: App-Passwort erforderlich (nicht normales Google-Passwort)

📧 Outlook/Hotmail

Server: smtp-mail.outlook.com
Port: 587
Verschlüsselung: STARTTLS
Hinweis: App-Passwort für Zwei-Faktor-Authentifizierung

📧 Firmen-Exchange-Server

Server: mail.ihrefirma.de
Port: 587 oder 25
Verschlüsselung: STARTTLS oder None
Hinweis: Kontaktieren Sie Ihre IT-Abteilung

Praktische Anwendungsfälle

🔧 Gmail-Einrichtung mit App-Passwort

  1. Google-Konto vorbereiten: Zwei-Faktor-Authentifizierung aktivieren
  2. App-Passwort erstellen: Google Konto → Sicherheit → App-Passwörter
  3. HCC Plan konfigurieren:
    • Server: smtp.gmail.com
    • Port: 587
    • Benutzername: ihre.email@gmail.com
    • Passwort: [16-stelliges App-Passwort]
    • Verschlüsselung: STARTTLS
  4. Testen und speichern

🏢 Firmen-Exchange-Setup

  1. IT-Abteilung kontaktieren für SMTP-Server-Details
  2. Server-Daten erfragen:
    • SMTP-Server-Adresse
    • Port und Verschlüsselung
    • Authentifizierung erforderlich?
  3. Konfiguration eingeben und testen
  4. Bei Problemen: IT-Support für Firewall/Proxy-Einstellungen

Häufige Probleme & Lösungen

Problem: "Authentifizierung fehlgeschlagen"

Ursache: Falsches Passwort oder keine App-Passwort bei Gmail/Outlook

Lösung: Verwenden Sie App-Passwörter für Gmail/Outlook. Überprüfen Sie Benutzername und Passwort. Bei Firmen-E-Mail: IT-Abteilung kontaktieren.

Problem: "Verbindung zum Server fehlgeschlagen"

Ursache: Falscher Server oder Port, Firewall-Blockierung

Lösung: Überprüfen Sie Server-Adresse und Port. Bei Firmen-Netzwerken: Proxy/Firewall-Einstellungen prüfen.

Problem: "SSL/TLS-Fehler"

Ursache: Verschlüsselungseinstellung passt nicht zum Server

Lösung: Versuchen Sie verschiedene Verschlüsselungen: STARTTLS (Port 587), SSL (Port 465), None (Port 25).

Best Practices

🔐 Sichere Konfiguration

Verwenden Sie immer verschlüsselte Verbindungen (STARTTLS oder SSL). Nutzen Sie starke Passwörter oder App-Passwörter. Ändern Sie Passwörter regelmäßig.

⚡ Verbindung testen

Testen Sie immer die Verbindung vor dem Speichern. Bei Fehlern: Schritt für Schritt die Einstellungen überprüfen. Dokumentieren Sie funktionierende Konfigurationen.

🏢 Firmen-Integration

Arbeiten Sie mit der IT-Abteilung zusammen für optimale Konfiguration. Beachten Sie Firmen-Policies zu E-Mail-Sicherheit. Dokumentieren Sie Einstellungen für andere Benutzer.

Erweiterte Funktionen

  • Verschlüsselte Passwort-Speicherung - Sichere lokale Speicherung der Anmeldedaten
  • Verbindungs-Test - Automatische Überprüfung der SMTP-Verbindung
  • Multi-Provider-Unterstützung - Funktioniert mit allen gängigen E-Mail-Anbietern
  • Konfigurationspersistenz - Einstellungen werden dauerhaft gespeichert

Technische Details

Die Email Configuration erfolgt über die EmailConfigDialog-Klasse mit TOML-basierter Konfigurationsspeicherung in configuration/email_config.toml. Die Passwörter werden verschlüsselt gespeichert und die Verbindung wird über Python's smtplib mit vollständiger TLS/SSL-Unterstützung getestet.
