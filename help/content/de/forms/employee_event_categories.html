Übersicht

Das Employee Event Kategorien-Management ist ein professionelles CRUD-Interface zur Verwaltung von Event-Kategorien für Mitarbeiter-Veranstaltungen. Der Dialog bietet ein geteiltes Layout mit Kategorien-Liste und Detail-Editor, umfassendes Usage-Tracking zur Verhinderung ungewollter Löschungen sowie vollständige Service-Layer-Integration mit Command Pattern-Unterstützung.

Hauptfunktionen

  • Split-Layout-Interface: Übersichtliche Trennung zwischen Kategorien-Liste (links) und Detail-Editor (rechts)
  • CRUD-Operationen: Vollständige Create, Read, Update, Delete-Funktionalität für Event-Kategorien
  • Usage-Tracking-System: Intelligente Anzeige der Verwendungshäufigkeit mit Schutz vor ungewollter Löschung
  • Dynamic UI-States: Adaptive Aktivierung/Deaktivierung des Detail-Bereichs basierend auf Kategorien-Auswahl
  • Real-time Validation: Sofortige Überprüfung auf Namens-Duplikate und Längenbegrenzungen (40 Zeichen)
  • Dark Theme UI: Moderne, augenfreundliche Benutzeroberfläche mit professionellem Styling
  • Service-Layer-Integration: EmployeeEventService mit ErrorResponseSchema für robuste Fehlerbehandlung

Grundbedienung

  1. Dialog öffnen: Zugang über das Employee Event Management-System
  2. Kategorien durchsuchen: Klicken Sie auf Kategorien in der linken Liste um Details anzuzeigen
  3. Neue Kategorie erstellen:
    • Klicken Sie "➕ New" unter der Kategorien-Liste
    • Der Detail-Editor wird aktiviert und das Name-Feld fokussiert
    • Geben Sie Namen und optional Beschreibung ein
    • Klicken Sie "💾 Save Category" zum Speichern
  4. Kategorie bearbeiten:
    • Wählen Sie eine Kategorie aus der Liste
    • Bearbeiten Sie Name oder Beschreibung im Detail-Editor
    • Klicken Sie "💾 Save Category" zum Übernehmen der Änderungen
  5. Kategorie löschen:
    • Wählen Sie die zu löschende Kategorie
    • Klicken Sie "🗑️ Delete"
    • Bestätigen Sie die Löschung in der Sicherheitsabfrage
  6. Kategorie auswählen: Klicken Sie "Select Category" um die aktuell gewählte Kategorie zu übernehmen

Tastaturkürzel

  • F1: Diese Hilfe öffnen
  • Tab: Zwischen Liste und Detail-Editor navigieren
  • Pfeiltasten: In der Kategorien-Liste navigieren
  • Enter: Ausgewählte Kategorie übernehmen (entspricht "Select Category")
  • Escape: Dialog abbrechen
  • Ctrl+N: Neue Kategorie erstellen (entspricht "➕ New")
  • Ctrl+S: Aktuelle Kategorie speichern (entspricht "💾 Save")
  • Delete: Kategorie löschen (entspricht "🗑️ Delete")
  • F2: Namen der ausgewählten Kategorie bearbeiten

Anwendungsbereich

Das Kategorien-Management ist fundamental für die Organisation von Employee Events. Kategorien ermöglichen eine strukturierte Klassifikation von Veranstaltungen und erleichtern sowohl die Planung als auch die Auswertung von Mitarbeiter-Events.

Typische Verwendungsszenarien:

  • Event-Strukturierung: Kategorisierung von Veranstaltungen nach Typ (Training, Meeting, Social Event, etc.)
  • Abteilungs-spezifische Events: Kategorien für verschiedene Unternehmensbereiche oder Teams
  • Compliance-Management: Kategorien für Pflicht-Schulungen, freiwillige Events, etc.
  • Berichtswesen: Strukturierte Auswertungen und Statistiken basierend auf Event-Kategorien

Benutzeroberfläche

Split-Layout-Design

Der Dialog verwendet ein modernes Split-Layout mit zwei Hauptbereichen:

  • Linker Bereich (Categories): Liste aller verfügbaren Kategorien mit Usage-Information
  • Rechter Bereich (Category Details): Editor für Name und Beschreibung der ausgewählten Kategorie

Kategorien-Liste (Linker Bereich)

Die Kategorien-Liste zeigt alle verfügbaren Kategorien in alphabetischer Sortierung:

  • Format: "Kategoriename (X events)" für verwendete Kategorien
  • Ungenutzte Kategorien: "Kategoriename (unused)" für nicht verwendete Kategorien
  • Placeholder: "No categories found" wenn keine Kategorien existieren
  • Visuelle Hervorhebung: Ausgewählte Kategorie wird farblich markiert

Aktions-Buttons (Unter Liste)

  • "➕ New": Aktiviert den Detail-Editor für eine neue Kategorie
  • "🗑️ Delete": Löscht die ausgewählte Kategorie (mit Usage-Validation)

Category Details (Rechter Bereich)

Der Detail-Editor ermöglicht die Bearbeitung von Kategorien-Eigenschaften:

  • Name-Feld: Eingabefeld mit 40-Zeichen-Limit und Placeholder-Text
  • Beschreibung-Feld: Mehrzeiliger Text-Editor für ausführliche Beschreibungen
  • Usage-Info: Dynamische Anzeige der Verwendung in Events
  • "💾 Save Category"-Button: Speichert Änderungen mit Validation

Dynamic UI-Verhalten

Die Benutzeroberfläche passt sich intelligent dem aktuellen Zustand an:

  • Deaktivierter Zustand: Detail-Editor ist deaktiviert wenn keine Kategorie ausgewählt ist
  • Aktivierter Zustand: Detail-Editor wird aktiviert sobald eine Kategorie gewählt oder "New" geklickt wird
  • Edit-Mode: Name-Feld wird automatisch fokussiert bei neuen Kategorien

Häufige Probleme & Lösungen

Problem: "Kategorie kann nicht gelöscht werden"

Ursache: Die Kategorie wird in einem oder mehreren Events verwendet

Lösung: Entfernen Sie zunächst die Kategorie aus allen verwendenden Events, dann können Sie sie löschen. Die Usage-Info zeigt die Anzahl der betroffenen Events

Problem: "Name bereits vorhanden"-Fehler

Ursache: Eine Kategorie mit dem eingegebenen Namen existiert bereits

Lösung: Wählen Sie einen eindeutigen Namen oder bearbeiten Sie die bestehende Kategorie mit ähnlichem Namen

Problem: "Detail-Editor bleibt deaktiviert"

Ursache: Keine Kategorie ausgewählt oder Placeholder-Item ("No categories found") angeklickt

Lösung: Klicken Sie auf eine echte Kategorie oder nutzen Sie "➕ New" um eine neue zu erstellen

Problem: "Speichern schlägt fehl"

Ursache: Leer-Name, zu langer Name (>40 Zeichen) oder Service-Layer-Fehler

Lösung: Überprüfen Sie Name-Eingabe auf Korrektheit oder kontaktieren Sie den Administrator bei Service-Fehlern

Problem: "Liste zeigt veraltete Daten"

Ursache: Cache-Inkonsistenz nach externen Änderungen

Lösung: Schließen Sie den Dialog und öffnen Sie ihn erneut um aktuelle Daten zu laden

  • Aussagekräftige Beschreibungen: Nutzen Sie das Beschreibungsfeld für klare Abgrenzung ähnlicher Kategorien
  • Zukunftsorientiert planen: Denken Sie an spätere Auswertungen und Reports bei der Namensgebung

🚀 CRUD-Workflow-Optimierung

  • Batch-Erstellung: Erstellen Sie ähnliche Kategorien in einem Workflow für Konsistenz
  • Usage-Monitoring: Prüfen Sie regelmäßig die Usage-Counts um ungenutzte Kategorien zu identifizieren
  • Backup vor Änderungen: Bei größeren Umstrukturierungen dokumentieren Sie die aktuellen Kategorien
  • Schrittweise Migration: Ändern Sie etablierte Kategorien-Strukturen graduell, nicht abrupt

🏗️ UI-Navigation-Strategien

  • Split-Layout nutzen: Lassen Sie die Liste geöffnet während Sie Details bearbeiten
  • Tastatur-Navigation: Nutzen Sie Pfeiltasten für schnelles Durchsuchen der Kategorien
  • Fokus-Management: Das System setzt automatisch den Fokus auf relevante Felder
  • Visual Feedback: Achten Sie auf die farbliche Hervorhebung ausgewählter Items

🔄 Usage-Protection-System nutzen

  • Vor Löschung prüfen: Beachten Sie die Usage-Info bevor Sie Kategorien löschen
  • Migration planen: Bei notwendigen Löschungen migrieren Sie Events zu anderen Kategorien
  • Historische Daten: Überlegen Sie Archivierung statt Löschung für historische Auswertungen
  • Rollback-Strategie: Nutzen Sie Command Pattern Undo bei versehentlichen Änderungen

Erweiterte Funktionen

Service-Layer-Integration

Der Dialog nutzt den EmployeeEventService für alle Datenoperationen:

  • get_all_categories_by_project: Lädt alle Kategorien für das aktuelle Projekt
  • get_all_events: Lädt Events für Usage-Count-Berechnung
  • ErrorResponseSchema-Handling: Strukturierte Fehlerbehandlung mit User-freundlichen Meldungen
  • Automatische Sortierung: Kategorien werden alphabetisch nach Name sortiert

Command Pattern Implementation

Alle Änderungen verwenden Command Pattern für Undo/Redo-Funktionalität:

  • category_commands.Create: Erstellung neuer Kategorien mit Validation
  • category_commands.Update: Aktualisierung bestehender Kategorien
  • category_commands.Delete: Sichere Löschung mit Usage-Validation
  • ContrExecUndoRedo: Vollständige Undo/Redo-Unterstützung für alle Operationen

Usage-Tracking-Algorithm

Intelligentes System zur Verfolgung der Kategorien-Verwendung:

  • Count-Berechnung: Dynamische Berechnung basierend auf employee_event_categories-Zuordnungen
  • Real-time Updates: Usage-Counts werden bei jeder Dialog-Öffnung neu berechnet
  • Protection-Logic: Verwendete Kategorien erhalten Lösch-Schutz mit Benutzer-Warnung
  • Dictionary-Caching: Effiziente Speicherung der Usage-Counts für Performance

Dynamic UI State Management

Adaptive Benutzeroberfläche mit intelligenter Zustandssteuerung:

  • _set_details_enabled: Zentrale Steuerung der Detail-Editor-Verfügbarkeit
  • Automatic Focus: Name-Feld wird bei neuen Kategorien automatisch fokussiert
  • Selection Persistence: Kategorien-Auswahl bleibt auch nach Operationen erhalten
  • Clear on Disable: Felder werden automatisch geleert wenn deaktiviert

Technische Details

Data Schema Integration

Der Dialog arbeitet mit strukturierten Data Schemas:

  • CategoryDetail: Vollständige Kategorie-Informationen mit ID, Name, Beschreibung
  • CategoryCreate: Schema für neue Kategorien mit Projekt-Zuordnung
  • CategoryUpdate: Schema für Kategorie-Aktualisierungen
  • ErrorResponseSchema: Strukturierte Fehler-Responses mit Nachricht und Details

Project-scoped Data Management

Kategorien sind projekt-spezifisch isoliert:

  • Project ID Binding: Alle Operationen sind auf das aktuelle Projekt beschränkt
  • Isolation: Kategorien verschiedener Projekte beeinflussen sich nicht
  • Scoped Service Calls: Alle Service-Aufrufe enthalten die Projekt-ID

UI Styling System

Professionelles Dark Theme mit modernen UI-Elementen:

  • Color Palette: Dunkle Grundfarben (#2d2d2d) mit Teal-Akzenten (#006d6d)
  • Interactive Elements: Hover-Effekte und Focus-States für bessere UX
  • Semantic Colors: Verschiedene Button-Farben für verschiedene Aktions-Typen
  • Responsive Design: Layout passt sich verschiedenen Dialog-Größen an

Error Handling & Logging

Umfassende Fehlerbehandlung und Logging:

  • Exception Catching: Try/Catch-Blöcke um alle Service-Aufrufe
  • User-friendly Messages: Benutzerverständliche Fehlermeldungen statt technischer Details
  • Structured Logging: Detaillierte Logs für Debugging und Monitoring
  • Graceful Degradation: System funktioniert auch bei partiellen Service-Fehlern
