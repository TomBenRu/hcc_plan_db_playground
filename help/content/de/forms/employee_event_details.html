Employee Event Details

Hilfe > Employee Event Details

Übersicht

Der Employee Event Details Dialog ist das zentrale Interface für die vollständige Verwaltung von Mitarbeiter-Events. Er unterstützt sowohl das Erstellen neuer Events als auch die umfassende Bearbeitung bestehender Events mit ausgeklügelter DateTime-Integration, Team- und Kategorie-Management, Teilnehmer-Auswahl sowie vollständiger Service-Layer-Anbindung und Command Pattern-Unterstützung.

Hauptfunktionen

  • Dual-Modus-Operation: Sowohl Create- als auch Update/Delete-Funktionalität in einem einheitlichen Interface
  • Intelligente DateTime-Integration: Getrennte Datum- und Zeit-Steuerelemente mit automatischer End-Zeit-Berechnung
  • Team-Assignment-System: Flexible Zuordnung zu einzelnen Teams oder allen Teams eines Projekts
  • Kategorie-Management: Integration des Kategorien-Systems mit direktem Zugang zur Kategorien-Verwaltung
  • Teilnehmer-Auswahl: Umfassendes Teilnehmer-Management mit datum-basierter Verfügbarkeitsprüfung
  • Adress-Integration: Vollständige Adress-Verwaltung mit Create/Edit-Funktionalität direkt im Dialog
  • Save-as-New-Feature: Option zum Duplizieren bestehender Events als Vorlage für neue Events
  • Command Pattern-Integration: Undo/Redo-Funktionalität für alle Änderungen

Grundbedienung

Neues Event erstellen

  1. Öffnen Sie den Dialog über "New Event" aus dem Employee Event Management
  2. Grundinformationen eingeben:
    • Geben Sie einen aussagekräftigen Titel ein (max. 40 Zeichen)
    • Verfassen Sie eine Beschreibung die Zweck und Inhalt des Events erklärt
  3. Datum und Zeit einstellen:
    • Start: Wählen Sie Datum und Uhrzeit des Event-Beginns
    • End: Das System setzt automatisch Ende = Start + 1 Stunde (anpassbar)
  4. Zuordnungen konfigurieren:
    • Teams: Wählen Sie beteiligte Teams oder "All teams"
    • Kategorie: Ordnen Sie das Event einer Kategorie zu oder verwalten Sie Kategorien direkt
    • Adresse: Wählen Sie einen Veranstaltungsort oder erstellen Sie eine neue Adresse
    • Teilnehmer: Klicken Sie "Select..." um spezifische Mitarbeiter einzuladen
  5. Klicken Sie "Create Event" zum Speichern

Bestehendes Event bearbeiten

  1. Öffnen Sie den Dialog über "Edit" bei einem bestehenden Event
  2. Alle Felder sind mit den aktuellen Event-Daten vorausgefüllt
  3. Nehmen Sie gewünschte Änderungen vor
  4. Speicher-Optionen:
    • "Update Event": Überschreibt das bestehende Event
    • "Save as new event": Aktivieren Sie die Checkbox um eine Kopie zu erstellen
  5. Alternativ: "🗑️ Delete Event" um das Event zu löschen

Tastaturkürzel

  • F1: Diese Hilfe öffnen
  • Tab: Zwischen Eingabefeldern navigieren
  • Enter: Event speichern (entspricht "Create/Update Event")
  • Escape: Dialog abbrechen
  • Ctrl+S: Event speichern (Alternative zu Enter)
  • Ctrl+D: Event duplizieren (aktiviert "Save as new event")
  • Ctrl+Delete: Event löschen (nur im Edit-Modus)
  • F2: Titel-Feld fokussieren
  • F3: Teilnehmer-Auswahl öffnen

Anwendungsbereich

Der Event Details Dialog ist das Herzstück des Employee Event Systems und ermöglicht die vollständige Konfiguration aller Event-Eigenschaften. Er wird sowohl für einzelne Ad-hoc-Events als auch für systematische Event-Serien verwendet.

Typische Einsatzszenarien:

  • Corporate Training Events: Schulungen mit spezifischen Teilnehmern und definierten Zeiträumen
  • Team Building Activities: Team-spezifische Events mit Location- und Kategorie-Zuordnung
  • Company-wide Meetings: Veranstaltungen für alle Teams mit umfassendem Teilnehmer-Management
  • Compliance Events: Pflicht-Schulungen mit präziser Dokumentation und Nachverfolgung

Benutzeroberfläche

General Information-Sektion

Der obere Bereich enthält die grundlegenden Event-Informationen:

  • Title-Feld: Kurzer, aussagekräftiger Event-Name (max. 40 Zeichen)
  • Description-Area: Mehrzeiliger Editor für detaillierte Event-Beschreibungen
  • Address-Selection: Dropdown mit bestehenden Adressen plus New/Edit-Buttons für Adress-Management

Date and Time-Sektion

Ausgeklügelte DateTime-Steuerung mit intelligenter Synchronisation:

  • Start-DateTime: Getrennte Date- und Time-Eingabe für Event-Beginn
  • End-DateTime: Automatische Berechnung (Start + 1 Stunde) mit manueller Übersteuerung
  • Intelligente Validation: End-Zeit muss immer nach Start-Zeit liegen
  • Minimum-Constraints: End-Datum/Zeit-Minimums passen sich automatisch an Start an
  • Multi-Day-Support: Unterstützung für mehrtägige Events

Assignments-Sektion

Umfassende Zuordnungs-Optionen für verschiedene Event-Typen:

  • Teams-Dropdown: Auswahl zwischen einzelnen Teams, "All teams" oder "No teams"

Options-Sektion (Edit-Modus)

Im Edit-Modus verfügbare erweiterte Optionen:

  • "Save as new event"-Checkbox: Erstellt eine Kopie statt das Original zu überschreiben
  • Button-Text-Updates: OK-Button ändert Text basierend auf der Save-as-new-Option

Dark Theme Interface

Professionelles Dark Theme mit modernen UI-Elementen:

  • Kontrast-reiche Farben: Optimiert für lange Arbeitszeiten und Augenkomfort
  • Gruppierte Bereiche: Visual grouping mit farbigen Überschriften
  • Interactive Feedback: Hover-Effekte und Focus-States für bessere Benutzerführung

Praktische Anwendungsfälle

🏢 Corporate Training Series

Szenario: Eine mehrteilige Schulungsreihe für verschiedene Abteilungen

Vorgehen:

  1. Verwenden Sie eindeutige Titel wie "Data Protection Training 2025"
  2. Kategorisieren Sie als "Mandatory Training"
  3. Nutzen Sie detaillierte Beschreibungen für Compliance-Dokumentation
  4. Wählen Sie alle betroffenen Teams systematisch aus

🔄 Event-Template-Erstellung

Szenario: Wiederkehrende Events mit ähnlicher Struktur

Vorgehen:

  1. Erstellen Sie ein "Template"-Event mit allen Standard-Einstellungen
  2. Verwenden Sie generische Titel wie "Monthly Team Meeting - Template"
  3. Im Edit-Modus: Aktivieren Sie "Save as new event"
  4. Passen Sie nur Datum und spezifische Details an
  5. So erstellen Sie schnell konsistente Event-Serien

Häufige Probleme & Lösungen

Problem: "End-Zeit kann nicht vor Start-Zeit gesetzt werden"

Ursache: Das System verhindert ungültige DateTime-Kombinationen automatisch

Lösung: Ändern Sie zuerst die Start-Zeit oder nutzen Sie die automatische End-Zeit-Berechnung (Start + 1 Stunde)

Problem: "Event kann nicht gespeichert werden - Validation Error"

Ursache: Pflichtfelder (Titel oder Beschreibung) sind leer

Lösung: Stellen Sie sicher, dass sowohl Titel als auch Beschreibung ausgefüllt sind

Problem: "Teilnehmer-Auswahl bleibt leer"

Ursache: Keine Mitarbeiter sind für das gewählte Datum verfügbar oder dem Team zugeordnet

Lösung: Überprüfen Sie Team-Zuordnungen für das Event-Datum oder wählen Sie ein anderes Datum

Problem: "Kategorien-Dropdown ist leer"

Ursache: Keine Event-Kategorien für das Projekt definiert

Lösung: Klicken Sie "Manage..." neben dem Kategorien-Dropdown um neue Kategorien zu erstellen

Problem: "Address Edit-Button deaktiviert"

Ursache: "No address" ausgewählt - keine editierbare Adresse vorhanden

Lösung: Wählen Sie eine bestehende Adresse aus dem Dropdown oder erstellen Sie eine neue mit "New..."

Best Practices

💡 Event-Strukturierung

  • Konsistente Titel: Verwenden Sie einheitliche Namens-Konventionen für ähnliche Event-Typen
  • Aussagekräftige Beschreibungen: Nutzen Sie das Beschreibungsfeld für Agenda, Ziele und wichtige Informationen
  • Kategorie-Systematik: Ordnen Sie Events konsequent passenden Kategorien zu
  • Template-Approach: Nutzen Sie "Save as new" für wiederkehrende Event-Strukturen

🚀 DateTime-Management-Strategien

  • Standard-Dauer nutzen: Die automatische 1-Stunden-Berechnung ist für die meisten Events passend
  • Pufferzeiten einplanen: Berücksichtigen Sie Setup- und Nachbereitungszeit in der Event-Dauer
  • Zeitzone-Konsistenz: Nutzen Sie die lokale Zeitzone-Integration für korrekte Anzeige
  • Multi-Day-Events: Bei mehrtägigen Events prüfen Sie die Auswirkungen auf Teilnehmer-Verfügbarkeit

🏗️ Team- und Teilnehmer-Management

  • Team-Hierarchie beachten: "All teams" für Company-weite Events, spezifische Teams für gezielte Veranstaltungen
  • Teilnehmer-Granularität: Kombinieren Sie Team-Auswahl mit individueller Teilnehmer-Selektion
  • Datum-abhängige Verfügbarkeit: Wählen Sie Teilnehmer basierend auf dem Event-Datum
  • Kapazitäts-Planung: Berücksichtigen Sie Location-Kapazitäten bei der Teilnehmer-Anzahl

🔄 Edit-Modus-Optimierungen

  • Save-as-new strategisch nutzen: Für Event-Serien oder ähnliche Events mit geänderten Details
  • Undo/Redo nutzen: Command Pattern ermöglicht sichere Experimente mit Rollback-Option
  • Backup-Strategie: Bei wichtigen Events vor großen Änderungen Save-as-new verwenden
  • Änderungs-Dokumentation: Nutzen Sie Beschreibungsfeld für Change-Log bei Updates

Erweiterte Funktionen

Intelligente DateTime-Synchronisation

Das System bietet ausgeklügelte DateTime-Management-Features:

  • Automatische End-Zeit-Berechnung: Start + 1 Stunde als Standard mit manueller Übersteuerung
  • Cross-Field-Validation: End-DateTime muss immer nach Start-DateTime liegen
  • Dynamic Minimums: End-Datum/Zeit-Minimums passen sich automatisch an Start an
  • Signal-Blocking: Verhindert Infinite-Update-Loops bei automatischen Anpassungen

Service-Layer-Integration

Vollständige Integration des EmployeeEventService:

  • CRUD-Operations: Create/Read/Update/Delete über strukturierte Service-Aufrufe
  • ErrorResponseSchema-Handling: Benutzerfreundliche Fehlerbehandlung
  • Data Caching: Effiziente Zwischenspeicherung von Teams, Kategorien und Adressen
  • Transactional Consistency: Atomare Operations mit Rollback-Fähigkeit

Multi-Dialog-Integration

Nahtlose Integration mit anderen Employee Event Dialogen:

  • Category Management: Direkter Zugang zur Kategorien-Verwaltung mit Rückkehr-Synchronisation
  • Participant Selection: Integration des Teilnehmer-Auswahl-Dialogs mit Datum-Kontextualisierung
  • Address Management: Eingebettete Adress-CRUD-Operationen
  • Cross-Dialog-State: Austausch von Auswahl-Zuständen zwischen Dialogen

Command Pattern Implementation

Vollständige Command Pattern-Unterstützung für alle Operations:

  • event_commands.Create: Event-Erstellung mit Schema-Validation
  • event_commands.Update: Event-Updates mit Differential-Logic
  • event_commands.Delete: Sichere Event-Löschung mit Abhängigkeits-Check
  • ContrExecUndoRedo: Undo/Redo-Stack für alle Dialog-Operationen

Technische Details

Event Schema Integration

Der Dialog arbeitet mit strukturierten Event Schemas:

  • EventDetail: Vollständige Event-Informationen für Edit-Modus
  • EventCreate: Schema für neue Events mit allen Zuordnungen
  • EventUpdate: Schema für Event-Updates mit ID-Referenz
  • ErrorResponseSchema: Strukturierte Fehler-Responses

Custom Widget Integration

Verwendung spezialisierter UI-Komponenten:

  • DateEditLocale: Lokalisierte Datum-Eingabe mit deutschen Formaten
  • TimeEditLocale: Lokalisierte Zeit-Eingabe mit 24-Stunden-Format
  • QComboBoxToFindData: Erweiterte ComboBox mit Such- und Data-Funktionalität

Dual-Mode Architecture

Intelligente Modus-Bestimmung und UI-Anpassung:

  • is_edit_mode / is_new_mode: Boolean-Flags für Modus-Bestimmung
  • Conditional UI Elements: Options-Sektion nur im Edit-Modus verfügbar
  • Dynamic Button Texts: OK-Button-Text passt sich an Modus und Optionen an
  • Mode-specific Workflows: Unterschiedliche Datenlade-Strategien je nach Modus

Data Caching & Performance

Optimierte Datenverwendung für bessere Performance:

  • teams_cache: Zwischenspeicherung aller Projekt-Teams
  • categories_cache: Cache für Event-Kategorien
  • addresses_cache: Lokale Kopie aller Projekt-Adressen
  • persons_cache: Mitarbeiter-Cache für Teilnehmer-Management
  • Smart Refresh: Selektive Cache-Updates nur bei tatsächlichen Änderungen
