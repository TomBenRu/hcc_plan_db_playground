Event-Planungsregeln

Hilfe > Event-Planungsregeln

Übersicht

Der Dialog "Event Planning Rules" ist das zentrale Werkzeug zur Definition automatischer Terminserien in HCC Plan. Mit diesem System können Sie komplexe, wiederkehrende Event-Strukturen für Standorte und Planungsperioden konfigurieren. Das Interface unterstützt multiple Regeln, verschiedene Tageszeiten, flexible Intervalle und intelligente Cast-Rule-Integration für eine vollautomatisierte Event-Erstellung.

Hauptfunktionen

  • Multi-Regel-System - Erstellung mehrerer parallel laufender Event-Serien mit individuellen Parametern
  • Flexible Starttag-Definition - Wahl zwischen festen Datumsauswahl oder wochentagsbasierten Berechnungen
  • Tageszeit-Integration - Auswahl aus verfügbaren Tageszeiten der Location-Plan-Period
  • Intervall-Steuerung - Konfigurierbare Abstände zwischen Events (täglich, wöchentlich, monatlich)
  • Wiederholungs-Management - Präzise Kontrolle über Anzahl und Häufigkeit der Events
  • Cast-Rule-Integration - Automatische Mitarbeiter-Zuweisung über vordefinierte Cast-Regeln
  • Same-Day-Grouping - Intelligente Gruppierung von Events am gleichen Tag
  • Persistente Speicherung - Speichern und Wiederverwenden von Regel-Konfigurationen
  • Validierung und Konflikt-Erkennung - Automatische Überprüfung auf zeitliche Überschneidungen
  • Event-Override-Schutz - Warnung bei Überschreibung bestehender Events

Grundbedienung

  1. Dialog öffnen - Zugriff über Planungsperioden-Verwaltung oder Location-Konfiguration
  2. Starttag-Modus wählen:
    • Festes Datum: Klicken Sie den Datum-Button zum Öffnen des Kalender-Dialogs
    • Wochentag-basiert: Wählen Sie Wochentag und "N-ter Tag"-Einstellung
  3. Regel-Parameter konfigurieren:
    • Wählen Sie "Time of Day" aus verfügbaren Optionen
    • Setzen Sie "Interval" für Abstände zwischen Events (Tage)
    • Definieren Sie "Repetitions" für Anzahl der Wiederholungen
    • Bestimmen Sie "Possible Count" für tatsächlich zu erstellende Events
  4. Optional: Zusätzliche Regeln hinzufügen:
    • Klicken Sie "New Rule" für weitere Event-Serien
    • Konfigurieren Sie jede Regel individuell
    • Nutzen Sie X-Button zum Entfernen unerwünschter Regeln
  5. Spezial-Optionen aktivieren:
    • "Same day selection" - Gruppiert alle Regeln auf gleiche Tage
    • "Rules for same day" - Wählt Cast-Rule für Events am gleichen Tag
  6. Konfiguration speichern/anwenden:
    • "Save Rules" - Speichert Konfiguration für spätere Wiederverwendung
    • "Reset Rules" - Löscht alle Regeln und startet neu
    • "OK" - Wendet Regeln an und erstellt Events

Tastaturkürzel

Praktische Anwendungsfälle

💡 Typische Regel-Konfigurationen

Tägliche Events: Interval=1, Repetitions=6 (Woche), Possible Count=5 (Mo-Fr)

Wöchentliche Termine: Start=Montag, Interval=7, Repetitions=4, Count=4 (monatlich)

Multiple Tageszeiten: Regel 1: Morgen, Regel 2: Nachmittag, gleiche Intervalle

Saisonale Events: Interval=14 (alle 2 Wochen), angepasst an Planungsperiode

Häufige Probleme & Lösungen

Problem: "Events with same time of day cannot be created twice on same day"

Ursache: Zwei Regeln versuchen Events zur gleichen Zeit am gleichen Tag zu erstellen.

Lösung: Ändern Sie Time-of-Day-Einstellungen oder Intervalle um Überschneidungen zu vermeiden.

Problem: "Same day selection" Checkbox bleibt deaktiviert

Ursache: Regeln haben unterschiedliche Start-Tage, Intervalle oder Event-Anzahlen.

Lösung: Harmonisieren Sie die Regel-Parameter - gleicher Start, gleiche Intervalle, gleiche Anzahl.

Problem: Repetitions-SpinBox begrenzt sich automatisch

Ursache: Events würden über das Ende der Planungsperiode hinausgehen.

Lösung: Reduzieren Sie Interval oder starten früher, oder erweitern die Planungsperiode.

Problem: Gespeicherte Regeln laden nicht korrekt

Ursache: Time-of-Day-Konfiguration der Location hat sich geändert seit Speicherung.

Lösung: System wählt automatisch nächste verfügbare Time-of-Day, Warnung wird angezeigt.

Problem: Cast-Rules werden nicht angewendet

Ursache: Cast-Rule ist nicht für das Projekt verfügbar oder wurde gelöscht.

Lösung: Prüfen Sie Cast-Rule-Verfügbarkeit oder erstellen neue mit "New Day Rule".

Best Practices

🚀 Effizienz-Optimierung

Regel-Minimierung: Verwenden Sie möglichst wenige Regeln - komplexe Single-Rules sind oft besser als viele Simple-Rules.

Template-System: Speichern Sie bewährte Konfigurationen und laden diese für ähnliche Standorte.

Validierung früh: Testen Sie Regel-Konfigurationen mit kurzen Perioden bevor Sie sie auf lange Zeiträume anwenden.

Cast-Rule-Vorbereitung: Definieren Sie Cast-Rules vor Event-Rules für seamless Integration.

🎯 Planungs-Strategien

Hierarchische Planung: Starten Sie mit Haupt-Events (wichtigste Zeit), fügen Neben-Events hinzu.

Flexibilität bewahren: Lassen Sie Raum für manuelle Anpassungen - nicht jeden Event automatisieren.

Seasonality berücksichtigen: Anpassung der Regeln an saisonale Schwankungen und Feiertage.

Performance-Awareness: Sehr viele Events können System-Performance beeinträchtigen.

Technische Details

Datenstrukturen

Das System verwendet sophisticated Data-Modeling:

  • RulesData Schema - first_day, time_of_day, interval, repeat, num_events
  • Rules Container - rules_data Liste, cast_rule_at_same_day, same_partial_days_flag
  • DefaultDict-Pattern - Dynamische Regel-Verwaltung mit automatischer Initialisierung
  • EventPlanningRules Persistence - Database-backed Konfigurationsspeicherung

UI-Architecture

Flexible, dynamische Benutzeroberfläche:

  • Grid-Layout-System - Dynamische Regel-Zeilen mit einheitlicher Spalten-Struktur
  • Widget-Factory-Pattern - Regel-spezifische Widget-Erstellung über Factory-Methods
  • Signal-Slot-Coordination - Complex Event-Handling zwischen UI-Komponenten
  • Partial-Function-Binding - Rule-Index-spezifische Event-Handler

Validation und Constraint-System

Umfassende Regel-Validierung:

  • Time-Conflict-Detection - Verhindert doppelte Events zur gleichen Zeit
  • Period-Boundary-Checking - Events müssen innerhalb Planungsperiode liegen
  • Logical-Consistency-Validation - Possible Count ≤ Repetitions + 1
  • Cross-Rule-Compatibility - Same-Day-Grouping-Requirements

Command-Pattern-Integration

Undo/Redo-System für komplexe Operations:

  • Controller-Based-Operations - ContrExecUndoRedo für Cast-Rule-Erstellung
  • Stack-Merging - Integration von Sub-Dialog-Operations
  • Rollback-Capability - Vollständige Rückgängigmachung bei Cancel
  • Transactional-Integrity - Atomic Operations für Event-Erstellung
