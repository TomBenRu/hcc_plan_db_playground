Fixed Cast Builder

Hilfe > Fixed Cast Builder

Übersicht

Der Fixed Cast Builder ermöglicht die präzise Definition fester Besetzungen für verschiedene Planungsebenen. Mit dem dynamischen Grid-System können Sie komplexe Cast-Strukturen mit logischen Operatoren (UND/ODER) erstellen und dabei flexible Besetzungsregeln definieren.

Hauptfunktionen

  • Dynamisches Grid - Flexibles Zeilen- und Spalten-System für Cast-Definition
  • Akteur-Auswahl - Dropdown-Listen aller verfügbaren Team-Mitglieder
  • Logische Operatoren - UND/ODER-Verknüpfungen zwischen Akteuren und Zeilen
  • Datum-basierte Planung - Berücksichtigung zukünftiger Personaländerungen
  • Automatische Vereinfachung - Optimierung komplexer Cast-Strukturen
  • Undo/Redo-System - Vollständige Rückgängig-Funktionalität

Grundbedienung

Erste Cast-Definition erstellen

  1. Klicken Sie auf das Plus-Symbol um die erste Zeile zu erstellen
  2. Wählen Sie einen Akteur aus der Dropdown-Liste
  3. Nutzen Sie Plus-Buttons um weitere Akteure hinzuzufügen
  4. Verwenden Sie Operatoren um Beziehungen zu definieren
  5. Fügen Sie weitere Zeilen für alternative Besetzungen hinzu

Operatoren verstehen

  1. UND (innerhalb Zeile) - Alle Akteure müssen anwesend sein
  2. ODER (innerhalb Zeile) - Einer der Akteure reicht aus
  3. Zeilen-Operatoren - Verknüpfung verschiedener Cast-Varianten
  4. Automatische Logik - System erstellt optimierte Ausdrücke

Tastaturkürzel

  • F1 - Diese Hilfe öffnen
  • Tab - Navigation zwischen Dropdown-Feldern
  • Strg+Z - Undo (rückgängig)
  • Strg+Y - Redo (wiederholen)
  • Strg+S - Speichern
  • Escape - Dialog abbrechen
  • Strg+N - Neue Zeile hinzufügen

Verwandte Funktionen

  • Cast-Gruppen-Eigenschaften - Integration mit Cast-Gruppen
  • Cast-Gruppen - Hierarchische Cast-Verwaltung
  • Team-Management - Verwaltung verfügbarer Akteure

Anwendungsbereich

Der Fixed Cast Builder wird verwendet für:

  • Feste Besetzungen - Definition unveränderlicher Cast-Zuordnungen
  • Flexible Alternativen - ODER-Verknüpfungen für alternative Besetzungen
  • Team-Kombinationen - UND-Verknüpfungen für Besetzungsgruppen
  • Datum-spezifische Planung - Berücksichtigung von Personaländerungen

Benutzeroberfläche

Grid-Bereich (Haupt-Arbeitsfeld)

  • Plus-Symbol (links) - Neue Zeile für alternative Cast-Varianten
  • Akteur-Dropdowns - Auswahl verfügbarer Team-Mitglieder
  • Plus/Minus-Buttons - Akteure hinzufügen/entfernen
  • Operator-Dropdowns - UND/ODER-Verknüpfungen zwischen Elementen

Datum-Bereich

  • Datum-Feld - Stichtag für Personalplanung
  • Automatische Berücksichtigung - Zukünftige Team-Änderungen
  • Verfügbarkeits-Warnung - Information über nicht verfügbare Personen

Aktions-Buttons

  • Reset - Dropdown-Menü für verschiedene Reset-Optionen
  • Undo - Letzte Änderung rückgängig machen
  • Redo - Rückgängig gemachte Änderung wiederholen
  • Speichern - Fixed Cast übernehmen und Dialog schließen
  • Abbrechen - Alle Änderungen verwerfen

Praktische Anwendungsfälle

Einfache Fixed Cast Definition

  1. Klicken Sie das Plus-Symbol für die erste Zeile
  2. Wählen Sie den gewünschten Akteur aus
  3. Für mehrere Akteure: Klicken Sie den Plus-Button rechts
  4. Wählen Sie den Operator (UND/ODER)
  5. Wählen Sie den zweiten Akteur
  6. Speichern Sie die Definition

Komplexe Cast-Strukturen (Mehrere Varianten)

  1. Erstellen Sie die erste Cast-Variante wie oben
  2. Klicken Sie das Plus-Symbol für eine neue Zeile
  3. Wählen Sie den Zeilen-Operator (UND/ODER)
  4. Definieren Sie die alternative Cast-Variante
  5. Wiederholen Sie für weitere Varianten
  6. System vereinfacht automatisch bei der Speicherung

Cast mit Datum-Abhängigkeit

  1. Wählen Sie das gewünschte Planungs-Datum
  2. System lädt automatisch verfügbare Personen für dieses Datum
  3. Erstellen Sie den Cast mit den verfügbaren Akteuren
  4. Beachten Sie Warnungen über Personal-Verfügbarkeit

Häufige Probleme & Lösungen

⚠️ Akteure nicht verfügbar

Problem: Ausgewählte Personen erscheinen nicht in der Liste
Ursachen:

  • Personen sind zum gewählten Datum nicht im Team
  • Team-Zuordnung ist fehlerhaft
  • Datum liegt außerhalb der Mitgliedschaftsdauer

Lösung: Überprüfen Sie Team-Mitgliedschaften und wählen Sie ein anderes Datum

⚠️ Cast zu komplex / nicht speicherbar

Problem: Definition kann nicht gespeichert werden
Ursachen:

  • Widersprüchliche Operatoren-Logik
  • Zu viele verschachtelte Ebenen
  • Logik-Fehler in der Struktur

Lösung: Nutzen Sie Reset → Löschen für Neustart oder vereinfachen Sie die Struktur schrittweise

⚠️ Anzahl Mitarbeiter überschritten

Problem: Fixed Cast benötigt mehr Personen als geplant
Lösungen:

  • Erhöhen Sie die Anzahl Akteure in der Planungsebene
  • Verwenden Sie ODER-Operatoren für Flexibilität
  • Vereinfachen Sie die Fixed Cast Definition

⚠️ Undo/Redo funktioniert nicht

Problem: Rückgängig-Funktion reagiert nicht
Ursachen:

  • Undo-Stack ist leer (nach Speicherung)
  • Maximale Anzahl von Änderungen erreicht
  • Grid-System in inkonsistentem Zustand

Lösung: Verwenden Sie Reset-Menü für vollständigen Neustart

Best Practices

🎯 Cast-Design-Empfehlungen

  • Einfachheit: Beginnen Sie mit einfachen Strukturen und erweitern Sie schrittweise
  • Redundanz vermeiden: Nutzen Sie System-Vereinfachung
  • Flexibilität: ODER-Verknüpfungen für alternative Besetzungen
  • Hierarchie beachten: Cast-Gruppen erben von übergeordneten Definitionen

🚀 Effizienz-Tipps

  • Operator-Logik: ODER-Operatoren erhöhen Flexibilität, UND-Operatoren Präzision
  • Datum-Strategie: Wählen Sie ein repräsentatives Datum für die Planung
  • Minimale Komplexität: Weniger Zeilen = bessere Performance
  • Operator-Effizienz: System optimiert Ausdrücke automatisch

💡 Mehrschicht-Logik

  • Basis-Team: Erste Zeile mit Kern-Besetzung (UND-Operatoren)
  • Alternativen: Weitere Zeilen mit ODER-Verknüpfung
  • Flexibilität: Mischung aus festen und variablen Komponenten

⚡ Performance bei großen Teams

  • Einschränkung: Nutzen Sie Datum-Filter für relevante Personen
  • Gruppierung: Definieren Sie Cast-Gruppen vor Fixed Cast
  • Hierarchie: Nutzen Sie Vererbung anstatt Redundanz

Erweiterte Funktionen

Cast-Logik verstehen

Operator-Bedeutungen

  • UND (innerhalb Zeile) - "Person A UND Person B" = beide müssen verfügbar sein
  • ODER (innerhalb Zeile) - "Person A ODER Person B" = eine der beiden reicht
  • UND (zwischen Zeilen) - "Zeile 1 UND Zeile 2" = beide Bedingungen erfüllen
  • ODER (zwischen Zeilen) - "Zeile 1 ODER Zeile 2" = eine Bedingung reicht

Beispiel-Strukturen

  • Einfach - "Max Mustermann" = genau diese Person
  • Alternative - "Max Mustermann ODER Anna Schmidt" = eine von beiden
  • Team - "Max Mustermann UND Anna Schmidt" = beide zusammen
  • Komplex - "(Max UND Anna) ODER (Klaus UND Maria)" = ein Team von beiden Varianten

Automatische Vereinfachung

System-Optimierung

  • Logik-Vereinfachung - Redundante Operatoren werden entfernt
  • Struktur-Optimierung - Komplexe Ausdrücke werden vereinfacht
  • Validierung - Widersprüchliche Definitionen werden erkannt

Beispiel-Vereinfachungen

  • (A UND A) → A - Doppelte Personen werden entfernt
  • (A ODER A) → A - Redundante Alternativen werden vereinfacht
  • ((A)) → A - Überflüssige Klammern werden entfernt
