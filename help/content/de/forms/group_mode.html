Gruppenmodus

Hilfe > Gruppenmodus

Übersicht

Der Gruppenmodus ist ein mächtiges Tool zur Organisation und Verwaltung von zusammengehörigen Terminen und Verfügbarkeiten. Er ermöglicht es, komplexe Planungsszenarien effizient zu verwalten, indem ähnliche Elemente in Gruppen zusammengefasst und gemeinsam bearbeitet werden.

Hauptfunktionen

  • Hierarchische Gruppenstruktur - Erstellen Sie verschachtelte Gruppenebenen
  • Drag & Drop - Verschieben Sie Termine zwischen Gruppen
  • Multi-Selection - Bearbeiten Sie mehrere Elemente gleichzeitig
  • Prioritäts-Management - Setzen Sie Gewichtungen für Gruppen
  • Anzahl-Kontrolle - Bestimmen Sie, wie viele Elemente aus einer Gruppe verwendet werden
  • Kontextmenüs - Schnelle Aktionen über Rechtsklick

Grundbedienung

Neue Gruppe erstellen

  1. Klicken Sie auf New Group
  2. Eine neue Gruppe wird erstellt und im Tree angezeigt
  3. Doppelklicken Sie auf die Gruppe, um deren Eigenschaften zu bearbeiten

Elemente zwischen Gruppen verschieben

  1. Einzelnes Element - Ziehen Sie das Element per Drag & Drop auf die Zielgruppe
  2. Mehrere Elemente - Wählen Sie mehrere Elemente aus (Strg+Klick) und verwenden Sie Drag & Drop
  3. Kontextmenü - Rechtsklick → In neue Gruppe verschieben oder In bestehende Gruppe verschieben

Gruppeneigenschaften bearbeiten

  1. Doppelklick auf eine Gruppe öffnet den Eigenschaften-Dialog
  2. Passen Sie die Anzahl der zu verwendenden Elemente an
  3. Setzen Sie Prioritäten für Untergruppen
  4. Konfigurieren Sie Mindestanforderungen (bei Verfügbarkeits-Gruppen)

Tastaturkürzel

  • F1 - Diese Hilfe öffnen
  • Strg+Klick - Multi-Selection (mehrere Elemente auswählen)
  • Shift+Klick - Bereich auswählen
  • Enter - Eigenschaften-Dialog öffnen (auf Gruppen)
  • Delete - Gruppe löschen (nur leere Gruppen)
  • Escape - Dialog schließen ohne Speichern

Verwandte Funktionen

  • Gruppeneigenschaften - Konfiguration von Gruppenanzahl und Prioritäten
  • Erweiterte Gruppeneigenschaften - Mit Verfügbarkeits-Bedingungen
  • Verfügbarkeitsplanung - Aufruf über "Switch to Group Mode"

Anwendungsbereich

Der Gruppenmodus wird in zwei Hauptbereichen verwendet:

  • Verfügbarkeitsplanung - Gruppierung von Mitarbeiter-Verfügbarkeiten (AvailDay Groups)
  • Event-Planung - Gruppierung von Standort-Events (Event Groups)

Benutzeroberfläche

Tree-Widget (Hauptbereich)

Das Tree-Widget zeigt die hierarchische Gruppenstruktur:

  • Gruppen - Angezeigt als "Group_XX" mit Hintergrundfarbe
  • Termine/Verfügbarkeiten - Einzelne Elemente mit Datum und Zeit
  • Spalten - Description, Date, Time of Day, Possible Count, Priority
  • Sortierung - Automatisch nach Datum sortiert

Steuerelemente

  • Edit Main Group - Bearbeitung der Hauptgruppen-Eigenschaften
  • New Group - Erstellt eine neue Untergruppe
  • Remove Group - Löscht die ausgewählte Gruppe (nur leere Gruppen)
  • Save/Cancel - Speichern oder Verwerfen der Änderungen

Farbkodierung

  • Gruppen-Hintergrund - Spezielle Hintergrundfarbe für Gruppenelemente
  • Datum-Spalte - Farblich hervorgehobene Datumsanzeige
  • Zeit-Spalte - Farblich hervorgehobene Zeitanzeige
  • Beschreibungs-Spalte - Unterscheidung zwischen Gruppen und Einzelelementen

Praktische Anwendungsfälle

Schichtgruppen erstellen

  1. Gruppenmodus aktivieren - Über "Switch to Group Mode" im Hauptformular
  2. Neue Gruppe erstellen - New Group klicken
  3. Termine zuweisen - Passende Termine per Drag & Drop in die Gruppe ziehen
  4. Eigenschaften setzen - Doppelklick auf Gruppe für Konfiguration
  5. Weitere Gruppen - Wiederholen für verschiedene Schichttypen

Wochenend-Verfügbarkeiten gruppieren

  1. Wochenend-Termine auswählen - Strg+Klick für Multi-Selection
  2. Rechtsklick - In neue Gruppe verschieben
  3. Priorität setzen - Doppelklick → Priorität auf "if necessary" (0)
  4. Anzahl begrenzen - Slider für maximale Anzahl Wochenend-Einsätze

Standort-spezifische Anforderungen

  1. Verfügbarkeits-Gruppe erstellen - Für bestimmte Mitarbeiter-Gruppe
  2. Mindestanforderungen aktivieren - Bedingung aktivieren Checkbox
  3. Standorte auswählen - Nur für bestimmte Einrichtungen aktivieren
  4. Mindestanzahl festlegen - Slider für minimale Zuweisungen

Häufige Probleme & Lösungen

⚠️ Gruppe kann nicht gelöscht werden

Problem: Remove Group funktioniert nicht
Ursache: Die Gruppe enthält noch Unterelemente
Lösung: Verschieben Sie alle Unterelemente in andere Gruppen, bevor Sie die Gruppe löschen

⚠️ Drag & Drop funktioniert nicht

Problem: Elemente lassen sich nicht verschieben
Ursachen:

  • Versuch, auf ein Einzelelement (nicht-Gruppe) zu droppen
  • Versuch, Element in sich selbst zu verschieben
  • Versuch, Gruppe in ihre eigenen Untergruppen zu verschieben

Lösung: Verwenden Sie gültige Drop-Ziele (andere Gruppen oder Hauptgruppe)

⚠️ Solo-Child Warnung

Problem: Fehlermeldung "At least one group has only one appointment"
Ursache: Eine Gruppe enthält nur ein einziges Element
Lösung: Fügen Sie weitere Elemente zur Gruppe hinzu oder verschieben Sie das einzelne Element in eine andere Gruppe

⚠️ Mindestanforderungen nicht verfügbar

Problem: Mindestanforderungen-Widgets sind deaktiviert
Ursache: Nicht alle Unterelemente haben verfügbare Tage oder die Gruppe ist nicht geeignet
Lösung: Stellen Sie sicher, dass alle Gruppenelemente gültige Verfügbarkeiten haben

⚠️ Dialog lässt sich nicht schließen

Problem: Dialog kann nicht mit Save geschlossen werden
Ursache: Validierungsfehler in der Gruppenstruktur
Lösung: Korrigieren Sie Solo-Child-Gruppen und leere Gruppen vor dem Speichern

⚠️ Änderungen gehen verloren

Problem: Änderungen werden nicht übernommen
Ursache: Dialog wurde mit Cancel geschlossen oder Validierung fehlgeschlagen
Lösung: Verwenden Sie Save zum Speichern oder korrigieren Sie Validierungsfehler

⚠️ Performance bei großen Gruppen

Problem: Langsame Reaktion bei vielen Elementen
Lösung: Verwenden Sie kleinere Untergruppen und vermeiden Sie zu tiefe Hierarchien

Best Practices

🚀 Empfohlene Vorgehensweise

  • Logische Struktur - Gruppieren Sie nach sinnvollen Kriterien (Zeit, Standort, Typ)
  • Aussagekräftige Namen - Nutzen Sie die automatische Nummerierung und ergänzen Sie bei Bedarf Notizen
  • Prioritäten durchdacht setzen - Verwenden Sie das dreistufige System strategisch
  • Regelmäßige Bereinigung - Entfernen Sie ungenutzte Gruppen regelmäßig
  • Testen vor Speichern - Überprüfen Sie die Gruppenstruktur vor dem Speichern

⚡ Effizienz-Tipps

  • Multi-Selection nutzen - Bearbeiten Sie mehrere Elemente gleichzeitig
  • Kontextmenüs verwenden - Schnellere Bedienung als Drag & Drop
  • Hierarchien planen - Denken Sie die Gruppenstruktur vorher durch
  • Prioritäten standardisieren - Verwenden Sie konsistente Prioritäts-Schemata

💡 Prioritäts-Strategien

  • Gleichmäßige Verteilung - Alle Elemente auf "preferred" (1)
  • Bevorzugte Zeiten - Wichtige Termine auf "highly preferred" (2)
  • Fallback-Optionen - Zusätzliche Verfügbarkeiten auf "if necessary" (0)

🎯 Prioritäts-Verwendung

  • "if necessary" (0) - Für Backup-Optionen und Notfall-Verfügbarkeiten
  • "preferred" (1) - Für Standard-Verfügbarkeiten und normale Events
  • "highly preferred" (2) - Für bevorzugte Zeiten und wichtige Events

🏗️ Komplexe Strukturen

  • Hierarchische Planung - Nutzen Sie mehrere Gruppenebenen für komplexe Szenarien
  • Template-Gruppen - Erstellen Sie wiederverwendbare Gruppenstrukturen
  • Conditional Logic - Kombinieren Sie Mindestanforderungen mit Prioritäten
  • Saisonale Anpassungen - Verschiedene Gruppenstrukturen für verschiedene Zeiträume

🔄 Workflow-Optimierung

  • Vorbereitung - Planen Sie die Gruppenstruktur bevor Sie den Modus aktivieren
  • Schrittweise Erstellung - Bauen Sie komplexe Strukturen schrittweise auf
  • Testing - Testen Sie die Gruppenlogik mit kleinen Beispielen
  • Dokumentation - Nutzen Sie Notizen zur Dokumentation komplexer Gruppierungen

Erweiterte Funktionen

Multi-Selection

Bearbeiten Sie mehrere Elemente gleichzeitig:

  • Strg+Klick - Mehrere Elemente auswählen
  • Shift+Klick - Bereich auswählen
  • Drag & Drop - Alle ausgewählten Elemente verschieben
  • Kontextmenü - Bulk-Operationen auf alle ausgewählten Elemente

Kontextmenü-Optionen

Rechtsklick auf ausgewählte Elemente bietet folgende Optionen:

  • "In neue Gruppe verschieben" - Erstellt automatisch eine neue Gruppe und verschiebt alle ausgewählten Elemente dorthin
  • "In bestehende Gruppe verschieben" - Submenu mit allen verfügbaren Zielgruppen
  • Hierarchische Anzeige - Zeigt die Gruppenstruktur im Submenu

Drag & Drop Regeln

Beim Verschieben gelten folgende Regeln:

  • Nicht in sich selbst - Elemente können nicht in sich selbst verschoben werden
  • Nicht in eigene Kinder - Gruppen können nicht in ihre Untergruppen verschoben werden
  • Nur auf Gruppen - Termine/Verfügbarkeiten sind keine gültigen Drop-Ziele
  • Automatische Aktualisierung - Tree-Widget wird nach dem Drop automatisch aktualisiert

Gruppeneigenschaften-Dialog

Anzahl-Steuerung

Bestimmen Sie, wie viele Elemente aus einer Gruppe verwendet werden:

  • Slider - Wählen Sie eine spezifische Anzahl (1 bis Anzahl Unterelemente)
  • "Alle direkt untergeordneten Elemente" - Checkbox für unbegrenzte Nutzung
  • Dynamische Anpassung - Maximalwert passt sich an verfügbare Elemente an

Prioritäts-Gewichtung

Setzen Sie Prioritäten für Untergruppen und Einzelelemente:

  • 0 - "if necessary" - Niedrige Priorität, nur bei Bedarf verwenden
  • 1 - "preferred" - Standard-Priorität, bevorzugte Verwendung
  • 2 - "highly preferred" - Hohe Priorität, vorrangige Verwendung

Verfügbarkeits-Gruppen (spezielle Funktionen)

Mindestanforderungen

Für Verfügbarkeits-Gruppen stehen zusätzliche Optionen zur Verfügung:

  • Mindestanzahl Zuweisungen - Legen Sie fest, wie viele Zuweisungen mindestens erfolgen müssen
  • Standort-spezifisch - Beschränken Sie Anforderungen auf bestimmte Standorte
  • "Bedingung aktivieren" - Ein/Ausschalten der Mindestanforderungen

Standort-Auswahl

Konfigurieren Sie standort-spezifische Anforderungen:

  • Alle Standorte - Anforderung gilt für alle Arbeitsplätze
  • Spezifische Standorte - Aktivieren Sie Nur für bestimmte Einrichtungen
  • Rechtsklick-Auswahl - Schnelle Einzelauswahl über Rechtsklick auf Checkbox

Erweiterte Steuerung

Prioritäts-System

Das dreistufige Prioritätssystem ermöglicht flexible Planungsstrategien mit optimaler Ressourcennutzung.

Anzahl-Logik

Die Anzahl-Steuerung funktioniert folgendermaßen:

  • Begrenzte Anzahl - System wählt die besten X Elemente basierend auf Prioritäten
  • Alle Elemente - Alle Unterelemente werden berücksichtigt
  • Dynamische Anpassung - Maximalwert passt sich an verfügbare Elemente an
  • Prioritäts-basierte Auswahl - Höhere Prioritäten werden bevorzugt ausgewählt

Mindestanforderungen (nur Verfügbarkeits-Gruppen)

Spezielle Logik für Mitarbeiter-Verfügbarkeiten:

  • Mindestanzahl - Mitarbeiter müssen mindestens X Zuweisungen in dieser Gruppe haben
  • Standort-Filter - Anforderung gilt nur für ausgewählte Standorte
  • Planungslogik - Verhindert zu ungleichmäßige Verteilung von Zuweisungen

Praktische Anwendungsszenarien

Schicht-Management

Organisieren Sie verschiedene Schichttypen:

  • Früh-/Spät-/Nachtschichten - Separate Gruppen für verschiedene Schichtzeiten
  • Wochenend-Schichten - Eigene Gruppe mit niedriger Priorität
  • Überstunden-Pool - Gruppe für zusätzliche Verfügbarkeiten

Event-Kategorisierung

Strukturieren Sie Events nach Typ oder Wichtigkeit:

  • Haupt-Events - Hohe Priorität, feste Anzahl
  • Zusatz-Events - Mittlere Priorität, flexible Anzahl
  • Optional-Events - Niedrige Priorität, nur bei verfügbaren Ressourcen

Geografische Gruppierung

Organisieren Sie nach Standorten:

  • Hauptstandorte - Hohe Priorität, Mindestanforderungen
  • Außenstellen - Flexible Handhabung
  • Mobile Einsätze - Separate Gruppe mit speziellen Regeln

Technische Details

Datenmodell

Der Gruppenmodus arbeitet mit verschiedenen Datentypen:

  • AvailDayGroup - Gruppen für Mitarbeiter-Verfügbarkeiten
  • EventGroup - Gruppen für Standort-Events
  • Master Groups - Hauptgruppen als Container
  • Child Groups - Untergruppen mit spezifischen Eigenschaften

Signal-System

Automatische Synchronisation zwischen Komponenten:

  • Group Mode Signals - Benachrichtigung über Gruppenmodus-Änderungen
  • Date Object Signals - Updates für einzelne Termine/Verfügbarkeiten
  • Reload Signals - Aktualisierung der Anzeige nach Änderungen

Validierung

Das System führt automatische Validierungen durch:

  • Solo-Child Detection - Warnung bei Gruppen mit nur einem Element
  • Empty Group Cleanup - Automatisches Entfernen leerer Gruppen
  • Konsistenz-Prüfung - Überprüfung der Gruppenstruktur beim Speichern

Tree-Widget Details

Navigation

  • Expand/Collapse - Klicken Sie auf die Pfeile zum Auf-/Zuklappen
  • Sortierung - Klicken Sie auf Spaltenköpfe zum Sortieren
  • Multi-Selection - Strg+Klick oder Shift+Klick für Mehrfachauswahl
  • Doppelklick - Öffnet Eigenschaften-Dialog für Gruppen

Eigenschaften-Dialog

  • Anzahl-Slider - Bestimmt, wie viele Unterelemente verwendet werden
  • "Alle Elemente" Checkbox - Deaktiviert Anzahl-Begrenzung
  • Prioritäts-Slider - Für jedes Unterelement einzeln konfigurierbar
  • Scroll-Area - Bei vielen Unterelementen automatisch scrollbar

Support

Bei Problemen mit dem Gruppenmodus:

  • Prüfen Sie die Gruppenstruktur vor dem Speichern auf Solo-Child-Gruppen
  • Verwenden Sie gültige Drop-Ziele beim Drag & Drop
  • Testen Sie komplexe Gruppierungen mit kleinen Beispielen
  • Konsultieren Sie die verwandten Funktionen für spezifische Gruppeneigenschaften
