Gruppeneigenschaften

Hilfe > Gruppeneigenschaften

Übersicht

Der Dialog Gruppeneigenschaften ermöglicht die Konfiguration wichtiger Parameter für die Gruppen- und Terminplanung. Hier legen Sie fest, wie viele untergeordnete Gruppen oder Termine aus einer übergeordneten Gruppe verwendet werden sollen und wie diese priorisiert werden.

Hauptfunktionen

  • Anzahl-Steuerung - Bestimmung wie viele untergeordnete Elemente verwendet werden
  • Automatik-Option - "Alle untergeordneten Elemente" für dynamische Auswahl
  • Prioritäts-Gewichtung - Variation Weight für jedes untergeordnete Element
  • Flexible Anpassung - Slider-basierte Einstellung für präzise Kontrolle
  • Undo/Redo - Vollständige Rückgängigmachen-Funktionalität

Grundbedienung

Anzahl der untergeordneten Elemente festlegen

  1. Slider verwenden - Ziehen Sie den Anzahl-Slider auf die gewünschte Anzahl
  2. Automatik aktivieren - Markieren Sie Alle untergeordneten Elemente für dynamische Auswahl
  3. Wert überprüfen - Der aktuelle Wert wird rechts neben dem Slider angezeigt

Prioritäten einstellen

  1. Element auswählen - Jedes untergeordnete Element hat einen eigenen Prioritäts-Slider
  2. Gewichtung anpassen - Bewegen Sie den Slider zwischen den Gewichtungs-Stufen
  3. Feedback beachten - Die Beschriftung zeigt die aktuelle Priorität an

Tastaturkürzel

  • F1 - Diese Hilfe anzeigen
  • Enter - Änderungen speichern und Dialog schließen
  • Escape - Dialog abbrechen (alle Änderungen verwerfen)
  • Tab - Zwischen Bedienelementen wechseln
  • Leertaste - Checkbox umschalten
  • Pfeiltasten - Slider-Werte anpassen

Verwandte Funktionen

  • Gruppenmodus - Hauptfenster für Gruppenverwaltung und Strukturierung
  • Gruppeneigenschaften Verfügbarkeit - Erweiterte Verfügbarkeits-Bedingungen
  • Standortplanung - Event-Planung mit Cast-Management

Anwendungsbereich

Die Gruppeneigenschaften werden verwendet für:

  • Ressourcen-Optimierung - Kontrolle der Anzahl verwendeter Untergruppen
  • Prioritäts-Management - Gewichtung verschiedener Planungsoptionen
  • Flexibilität vs. Kontrolle - Balance zwischen automatischer und manueller Steuerung
  • Planungs-Algorithmus - Steuerung der automatischen Plan-Berechnung

Benutzeroberfläche

Anzahl der untergeordneten Gruppen/Termine

Count-Slider

  • Funktion - Bestimmt die exakte Anzahl der zu verwendenden untergeordneten Elemente
  • Bereich - 1 bis Anzahl der verfügbaren untergeordneten Elemente
  • Interaktion - Klicken oder Ziehen für Wertänderung

"Alle untergeordneten Elemente" Checkbox

  • Funktion - Aktiviert automatische Verwendung aller verfügbaren Elemente
  • Effekt - Deaktiviert den Count-Slider und setzt Anzahl dynamisch
  • Anwendung - Ideal für flexible Planungsszenarien

Prioritäten der untergeordneten Gruppen/Termine

Variation Weight Slider

Gewichtungs-Stufen:

  • 0 - Niedrige Priorität (wird seltener verwendet)
  • 1 - Normale Priorität (Standard-Gewichtung)
  • 2 - Hohe Priorität (wird bevorzugt verwendet)

Dialog-Steuerung

Speichern/Abbrechen

  • Speichern - Übernimmt alle Änderungen in die Planung
  • Abbrechen - Verwirft alle Änderungen und stellt ursprüngliche Werte wieder her
  • Undo-System - Automatische Rückgängigmachung bei Abbruch

Praktische Anwendungsfälle

Begrenzte Gruppenauswahl konfigurieren

  1. Öffnen Sie die Gruppeneigenschaften für eine übergeordnete Gruppe
  2. Stellen Sie den Count-Slider auf die gewünschte maximale Anzahl
  3. Passen Sie die Variation Weights der untergeordneten Elemente an
  4. Setzen Sie bevorzugte Elemente auf hohe Priorität (2)
  5. Speichern Sie die Einstellungen

Dynamische Vollauswahl

  1. Aktivieren Sie Alle untergeordneten Elemente
  2. Konfigurieren Sie die Prioritäten für optimale Verteilung
  3. Wichtige Elemente: Hohe Priorität (2)
  4. Standard-Elemente: Normale Priorität (1)
  5. Backup-Elemente: Niedrige Priorität (0)
  6. Speichern Sie die Konfiguration

Prioritäten-Balancing

  1. Analysieren Sie die aktuelle Verteilung der untergeordneten Elemente
  2. Identifizieren Sie über- oder unterrepräsentierte Gruppen/Termine
  3. Passen Sie die Variation Weights entsprechend an
  4. Überprüfen Sie die Auswirkungen durch Testplanung
  5. Verfeinern Sie bei Bedarf die Gewichtungen

Häufige Probleme & Lösungen

⚠️ Slider reagiert nicht

Problem: Count-Slider ist deaktiviert
Ursache: Alle untergeordneten Elemente ist aktiviert
Lösung: Deaktivieren Sie die Checkbox für manuelle Anzahl-Kontrolle

⚠️ Keine untergeordneten Elemente sichtbar

Problem: Dialog zeigt keine Prioritäts-Slider
Ursache: Gruppe hat keine untergeordneten Gruppen oder Termine
Lösung: Überprüfen Sie die Gruppenstruktur im Hauptfenster

⚠️ Änderungen werden nicht übernommen

Problem: Einstellungen sind nach Dialog-Schließung zurückgesetzt
Ursache: Dialog wurde abgebrochen statt gespeichert
Lösung: Verwenden Sie Speichern statt Abbrechen oder Escape

⚠️ Prioritäten scheinen keine Wirkung zu haben

Problem: Plan-Berechnung ignoriert Gewichtungen
Ursache: Alle Elemente haben dieselbe Gewichtung
Lösung: Differenzieren Sie die Variation Weights deutlicher (0, 1, 2)

Best Practices

🎯 Effiziente Prioritäts-Verteilung

  • Hohe Priorität (2): Maximal 20-30% der Elemente
  • Normale Priorität (1): 50-60% der Elemente
  • Niedrige Priorität (0): 20-30% für Flexibilität

🚀 Planungs-Optimierung

  • Testen Sie verschiedene Anzahl-Einstellungen für optimale Abdeckung
  • Berücksichtigen Sie saisonale Schwankungen bei der Gewichtung
  • Verwenden Sie Alle Elemente für maximale Flexibilität
  • Dokumentieren Sie bewährte Einstellungen für wiederkehrende Planungen

⚡ Effizienz-Tipps

  • Ändern Sie mehrere Prioritäten vor dem Speichern (Batch-Update)
  • Nutzen Sie die Scroll-Area bei vielen untergeordneten Elementen
  • Testen Sie Einstellungen mit kleinen Gruppen vor Vollproduktion

💡 Kernfunktion

Kontrolliert die Anzahl und Gewichtung von Untergruppen/Terminen bei der automatischen Planung.

Erweiterte Funktionen

Scroll-Area bei vielen Elementen

Bei vielen untergeordneten Elementen wird eine Scroll-Area angezeigt:

  • Automatische Aktivierung - Bei mehr als 8-10 Elementen
  • Scrolling - Mausrad oder Scrollbalken verwenden
  • Tastatur-Navigation - Tab-Taste für Navigation zwischen Elementen

Batch-Updates

Effiziente Bearbeitung mehrerer Prioritäten:

  • Mehrere Änderungen - Passen Sie alle gewünschten Prioritäten an
  • Einmalig speichern - Alle Änderungen werden zusammen übernommen
  • Konsistenz - Vermeidet Zwischenstände bei komplexeren Konfigurationen

Testmodus

Überprüfung der Einstellungen vor Produktiveinsatz:

  • Kleine Gruppen - Testen Sie Konfigurationen mit überschaubaren Gruppen
  • Planungsvorschau - Nutzen Sie die Plan-Berechnung für Test-Szenarien
  • Iterative Anpassung - Verfeinern Sie Einstellungen basierend auf Ergebnissen

Technische Details

Dialog-Implementierung

  • Dialog-Klasse - DlgGroupProperties
  • Hilfe-Aufruf - F1-Taste im Dialog
  • Parent-Fenster - Gruppenmodus-Hauptfenster
  • Datenmodell - Tree-basierte Gruppenhierarchie

Daten-Persistierung

  • Automatische Speicherung - Beim Bestätigen des Dialogs
  • Rollback-Mechanismus - Bei Abbruch werden ursprüngliche Werte wiederhergestellt
  • Konsistenz-Prüfung - Validierung der Eingaben vor Speicherung

Integration mit Plan-Berechnung

  • Gewichtungs-Algorithmus - Variation Weights beeinflussen Auswahlwahrscheinlichkeit
  • Anzahl-Beschränkung - Count-Wert limitiert verwendete Untergruppen
  • Dynamische Anpassung - Alle Elemente ermöglicht flexible Auswahl

Support

Bei Problemen mit den Gruppeneigenschaften:

  • Überprüfen Sie die Gruppenstruktur im Hauptfenster
  • Testen Sie Einstellungen mit einfachen Szenarien
  • Kontrollieren Sie die Auswirkungen über die Plan-Berechnung
  • Dokumentieren Sie erfolgreiche Konfigurationen für zukünftige Verwendung
