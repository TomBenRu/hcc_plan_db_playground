Erweiterte Gruppeneigenschaften

Hilfe > Erweiterte Gruppeneigenschaften

🎯 Übersicht

Der Dialog Erweiterte Gruppeneigenschaften bietet alle Funktionen der Standard-Gruppeneigenschaften plus spezielle Verfügbarkeits-Bedingungen. Diese erweiterte Version ermöglicht es, Mindestanforderungen für Zuweisungen zu definieren und standortspezifische Bedingungen zu setzen.

💡 Erweiterte Funktion: Kontrolliert nicht nur Anzahl und Prioritäten, sondern auch Mindestanforderungen für Mitarbeiterzuweisungen basierend auf Verfügbarkeits-Tagen.
⚠️ Wichtig: Diese erweiterten Features sind nur verfügbar, wenn alle untergeordneten Elemente Verfügbarkeits-Daten haben.

🔧 Hauptfunktionen

  • Standard-Gruppenfunktionen:
    • Anzahl-Steuerung für untergeordnete Elemente
    • Automatik-Option für alle Elemente
    • Prioritäts-Gewichtung (Variation Weight)
  • Verfügbarkeits-Bedingungen:
    • Mindestanzahl von Zuweisungen pro Mitarbeiter
    • Standortspezifische Bedingungen
    • Flexible Standort-Auswahl
    • Rechtsklick-Schnellauswahl für einzelne Standorte
  • Intelligente Anpassung:
    • Automatische Deaktivierung bei ungeeigneten Daten
    • Dynamische Maximalwert-Anpassung
    • Bedingungs-Validierung

📋 Grundlegende Bedienung

Standard-Gruppenkonfiguration (wie Basis-Version):

  1. Anzahl festlegen: Slider für gewünschte Anzahl untergeordneter Elemente
  2. Automatik nutzen: "Alle untergeordneten Elemente" für dynamische Auswahl
  3. Prioritäten setzen: Variation Weight Slider für jedes Element anpassen

Verfügbarkeits-Bedingungen konfigurieren:

  1. Bedingung aktivieren: Markieren Sie "Bedingung aktivieren?"
  2. Mindestanzahl einstellen: Verwenden Sie den Mindestanzahl-Slider
  3. Standorte auswählen (optional):
    • Aktivieren Sie "Nur für bestimmte Einrichtungen aktivieren"
    • Wählen Sie die gewünschten Standorte aus
    • Rechtsklick für Einzelauswahl
  4. Einstellungen speichern

🚀 Erweiterte Verfügbarkeits-Features

Mindestanzahl von Zuweisungen:

Zweck: Stellt sicher, dass Mitarbeiter nur zugewiesen werden, wenn sie eine Mindestanzahl von Zuweisungen erreichen

Anwendung: Verhindert ineffiziente Einzelzuweisungen

Bereich: 1 bis Maximum basierend auf verfügbaren Terminen

Standortspezifische Bedingungen:

Funktion: Bedingungen gelten nur für ausgewählte Einrichtungen

Flexibilität: Verschiedene Standorte können unterschiedliche Anforderungen haben

Rechtsklick-Feature: Schnelle Einzelauswahl eines Standorts

Automatische Anpassungen:

Intelligente Limits: Maximalwerte passen sich an verfügbare Termine an

Bedingungs-Validierung: System prüft Machbarkeit der Einstellungen

UI-Anpassung: Elemente werden bei Unverfügbarkeit deaktiviert

🖱️ Bedienelemente im Detail

Standard-Steuerung (inherited):

Anzahl untergeordneter Elemente

Count-Slider: Exakte Anzahl-Kontrolle

"Alle Elemente" Checkbox: Dynamische Vollauswahl

Variation Weight Slider: Prioritäts-Gewichtung (0=niedrig, 1=normal, 2=hoch)

Dynamischer Bereich: 1 bis berechnetes Maximum

Aktualisierung: Passt sich an Anzahl-Änderungen an

Effekt: Zeigt/versteckt Standort-Auswahl

Multi-Selection: Mehrere Standorte gleichzeitig wählbar

Rechtsklick: Schnelle Einzelauswahl

Tooltip: Zeigt verfügbare Aktionen an

🔄 Erweiterte Workflows

Workflow 1: Mindestanforderungen für alle Standorte

  1. Öffnen Sie die Gruppeneigenschaften für eine Verfügbarkeits-Gruppe
  2. Konfigurieren Sie Basis-Einstellungen (Anzahl, Prioritäten)
  3. Aktivieren Sie "Bedingung aktivieren?"
  4. Stellen Sie die Mindestanzahl ein (z.B. 3 Zuweisungen)
  5. Lassen Sie Standort-Spezifikation deaktiviert (gilt für alle)
  6. Speichern Sie die Konfiguration

Workflow 2: Standortspezifische Bedingungen

  1. Aktivieren Sie die Verfügbarkeits-Bedingung
  2. Aktivieren Sie "Nur für bestimmte Einrichtungen"
  3. Wählen Sie die relevanten Standorte aus der Liste
  4. Verwenden Sie Rechtsklick für schnelle Einzelauswahl
  5. Justieren Sie die Mindestanzahl je nach Standort-Anforderungen
  6. Testen Sie die Auswirkungen vor finaler Speicherung

Workflow 3: Optimierung bestehender Bedingungen

  1. Analysieren Sie aktuelle Zuweisungs-Patterns
  2. Identifizieren Sie Standorte mit Über-/Unterbesetzung
  3. Passen Sie Mindestanforderungen entsprechend an
  4. Modifizieren Sie Standort-Spezifikationen
  5. Testen Sie mit verschiedenen Anzahl-Einstellungen
  6. Dokumentieren Sie optimale Konfigurationen

⌨️ Tastaturkürzel

F1 - Diese erweiterte Hilfe anzeigen

Enter - Änderungen speichern und Dialog schließen

Esc - Dialog abbrechen (alle Änderungen verwerfen)

Tab - Zwischen allen Bedienelementen wechseln

Leertaste - Checkbox umschalten (auch für Standorte)

Pfeiltasten - Slider-Werte anpassen (Anzahl und Mindestanforderungen)

Strg+Click - Multi-Selection bei Standorten

🔧 Häufige Probleme & Lösungen

Problem: Verfügbarkeits-Features nicht verfügbar

Ursache: Nicht alle untergeordneten Elemente haben Verfügbarkeits-Daten

Lösung: Überprüfen Sie, ob alle Termine mit Verfügbarkeits-Informationen verknüpft sind

Problem: Mindestanzahl-Slider reagiert nicht

Ursache: "Bedingung aktivieren?" ist nicht markiert

Lösung: Aktivieren Sie zuerst die Verfügbarkeits-Bedingung

Problem: Standort-Auswahl nicht sichtbar

Ursache: "Nur für bestimmte Einrichtungen" ist deaktiviert

Lösung: Aktivieren Sie die standortspezifische Bedingung

Problem: Keine Standorte in der Liste

Ursache: Für die gewählten Termine sind keine Standorte definiert

Lösung: Überprüfen Sie die Standort-Konfiguration für die betroffenen Termine

Problem: Mindestanforderung zu hoch

Ursache: Wert überschreitet verfügbare Termine

Lösung: System begrenzt automatisch auf maximum verfügbare Termine

Problem: Änderungen werden nicht gespeichert

Ursache: Dialog wurde abgebrochen oder Validierung fehlgeschlagen

Lösung: Überprüfen Sie alle Bedingungen und verwenden Sie "Speichern"

⚡ Effizienz-Strategien

  • Batch-Konfiguration: Konfigurieren Sie alle Parameter vor dem Speichern
  • Template-Ansatz: Speichern Sie bewährte Konfigurationen als Referenz
  • Schrittweise Optimierung: Beginnen Sie mit einfachen Bedingungen
  • Monitoring: Überwachen Sie die Auswirkungen neuer Bedingungen

🔍 Fehlerdiagnose

  • Daten-Check: Überprüfen Sie Verfügbarkeits-Daten regelmäßig
  • Bedingungen testen: Nutzen Sie kleine Testgruppen
  • Standort-Verfügbarkeit: Prüfen Sie Standort-Termine-Zuordnungen
  • Performance: Komplexe Bedingungen können Planungszeit erhöhen

🔄 Komplexe Anwendungsszenarien

Szenario 1: Multi-Standort-Event mit Mindestanforderungen

  1. Situation: Event an 5 Standorten, jeder Mitarbeiter soll mindestens 3 Termine bekommen
  2. Konfiguration:
    • Alle untergeordneten Elemente aktivieren
    • Mindestanzahl auf 3 setzen
    • Standortspezifische Bedingungen deaktiviert (gilt für alle)
    • Prioritäten je nach Standort-Wichtigkeit setzen
  3. Resultat: Effiziente Verteilung mit garantierten Mindestarbeitszeiten

Szenario 2: Kritische Standorte mit höheren Anforderungen

  1. Situation: Hauptstandort braucht erfahrenere Mitarbeiter (mehr Termine)
  2. Konfiguration:
    • Mindestanforderungen aktivieren
    • Standortspezifische Bedingungen aktivieren
    • Nur Hauptstandort auswählen (Rechtsklick)
    • Höhere Mindestanzahl für Hauptstandort (z.B. 5)
    • Hohe Priorität für wichtige Termine
  3. Resultat: Qualifizierte Mitarbeiter werden bevorzugt dem Hauptstandort zugewiesen

Szenario 3: Flexible Backup-Planung

  1. Situation: Backup-Gruppe für Ausfälle, niedrige Mindestanforderungen
  2. Konfiguration:
    • Begrenzte Anzahl untergeordneter Elemente (z.B. 50%)
    • Niedrige Mindestanforderungen (1-2 Termine)
    • Niedrige bis normale Prioritäten
    • Standort-Flexibilität für alle Einrichtungen
  3. Resultat: Flexible Backup-Kapazität ohne Überplanungen

🔗 Integration mit anderen Modulen

Verbindung zu Verfügbarkeitsplanung:

Diese erweiterten Eigenschaften arbeiten eng mit der Mitarbeiterplanung zusammen:

  • Verfügbarkeits-Daten: Stammen aus der Mitarbeiterplanung
  • Mindestanforderungen: Beeinflussen Zuweisungsalgorithmen
  • Standort-Filter: Berücksichtigen Mitarbeiter-Standort-Präferenzen

Auswirkungen auf Event-Planung:

Konfigurationen wirken sich direkt auf die Standortplanung aus:

  • Cast-Größe: Mindestanforderungen beeinflussen Besetzungsgrößen
  • Standort-Prioritäten: Wichtige Standorte bekommen bevorzugte Behandlung
  • Ressourcen-Optimierung: Effizientere Mitarbeiter-Verteilung

🔗 Verwandte Funktionen

  • Basis-Gruppeneigenschaften: Standard-Version ohne Verfügbarkeits-Features
  • Gruppenmodus: Hauptfenster für Gruppenverwaltung
  • Mitarbeiterplanung: Verfügbarkeitsplanung für Personen
  • Standortplanung: Event-Planung mit Cast-Management
  • Team-Verwaltung: Grundlegende Team-Konfiguration

🔧 Technische Details

Dialog-Klasse: DlgGroupPropertiesAvailDay (erweitert DlgGroupProperties)

Hilfe-Aufruf: F1-Taste im Dialog

Parent-Fenster: Gruppenmodus-Hauptfenster

Datenmodell: Tree-basierte Gruppenhierarchie + Verfügbarkeits-Daten

Abhängigkeiten: RequiredAvailDayGroups, LocationOfWork
