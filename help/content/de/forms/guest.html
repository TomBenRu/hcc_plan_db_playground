Gast hinzufügen

Hilfe > Gast hinzufügen

Übersicht

Der Dialog "Gast hinzufügen" ermöglicht es Ihnen, externe Personen oder Gastpersonal zu einem Termin hinzuzufügen. Dies ist ein einfacher Name-Input-Dialog, der als Teil des Cast-Management-Systems verwendet wird, wenn reguläre Mitarbeiter nicht verfügbar sind oder zusätzliches Personal benötigt wird.

Hauptfunktionen

  • Einfache Namen-Eingabe - Schnelles Hinzufügen von Gast-Namen über ein Textfeld
  • Cast-System Integration - Nahtlose Einbindung in das Terminbesetzungs-System
  • Flexible Personalnamen - Freie Texteingabe ohne Einschränkungen für Name oder Bezeichnung
  • Sofortige Verfügbarkeit - Gast wird direkt nach Bestätigung zur Terminbesetzung hinzugefügt
  • Minimalistisches Design - Fokussiert auf die einzige notwendige Information: den Namen

Grundbedienung

  1. Dialog öffnet automatisch - Wird vom Cast-Management geöffnet, wenn "Gast" aus einer Mitarbeiter-Liste gewählt wird
  2. Namen eingeben:
    • Klicken Sie in das "Gast-Besetzung" Eingabefeld
    • Geben Sie den vollständigen Namen oder eine Bezeichnung ein
    • Verwenden Sie einen eindeutigen, verständlichen Namen
  3. Eingabe bestätigen:
    • Klicken Sie "OK" um den Gast zu erstellen
    • Oder drücken Sie Enter für schnelle Bestätigung
  4. Automatische Rückkehr - Sie kehren automatisch zum Cast-Management Dialog zurück

Tastaturkürzel

F1 - Diese Hilfe öffnen

Enter - Gast erstellen und Dialog schließen

Escape - Dialog ohne Gast-Erstellung schließen

Ctrl+A - Gesamten Text im Namen-Feld auswählen

Verwandte Funktionen

  • Termin Bearbeiten - Hauptdialog für Cast-Management, von dem aus Gäste hinzugefügt werden
  • Planungsübersicht - Hauptansicht zur Terminplanung und -verwaltung
  • Tagesverfügbarkeit - Übersicht der regulären Mitarbeiter-Verfügbarkeiten

Anwendungsbereich

🎯 Das Hinzufügen von Gästen wird verwendet für:

  • Extern beauftragte Personen - Freiberufler oder externe Dienstleister für spezielle Events
  • Krankheitsvertretung - Schnelle Lösung bei kurzfristigen Mitarbeiter-Ausfällen
  • Zusätzliche Kapazität - Verstärkung bei Events mit höherem Personalbedarf
  • Spezialisierte Rollen - Experten für spezifische Aufgaben, die nicht im regulären Team sind
  • Temporäres Personal - Aushilfen oder Praktikanten für bestimmte Termine

Benutzeroberfläche

Dialog-Aufbau

  • Titel "Gast-Besetzung" - Klarer Hinweis auf die Funktion des Dialogs
  • Erklärungstext - "Geben Sie den Namen des Gastes ein" als Anleitung
  • Eingabefeld "Gast-Besetzung" - Freies Textfeld für Namen oder Bezeichnung
  • Standard-Buttons - OK und Abbrechen für eindeutige Aktionsmöglichkeiten

Eingabefeld-Details

  • Freie Texteingabe - Keine Einschränkungen bei Zeichen oder Länge
  • Kein Standardwert - Feld ist initial leer für individuelle Eingabe
  • Sofortiger Fokus - Cursor steht bereits im Eingabefeld
  • Standard-Textbearbeitung - Kopieren, Einfügen, Auswählen möglich

Praktische Anwendungsfälle

💡 Typische Gast-Szenarien

Externe Expertise: "Dr. Schmidt (Gastdozent)" für Fachvorträge oder spezialisierte Workshops.

Kurzfristige Vertretung: "Maria (Aushilfe)" wenn reguläre Mitarbeiter kurzfristig ausfallen.

Event-spezifisch: "Fotograf Müller" für besondere Anlässe, die nicht zur regulären Besetzung gehören.

Praktikanten/Studenten: "Anna (Praktikantin)" für temporäre Unterstützung in Lernphasen.

Häufige Probleme & Lösungen

Problem: Dialog öffnet sich nicht

Ursache: "Gast"-Option wurde im Cast-Management nicht korrekt ausgewählt.

Lösung: Gehen Sie zurück zum Cast-Dialog und wählen Sie explizit "Gast" aus der Dropdown-Liste.

Problem: Name wird nicht übernommen

Ursache: Dialog wurde mit "Abbrechen" oder "X" geschlossen statt mit "OK".

Lösung: Verwenden Sie "OK" oder Enter-Taste um den Namen zu bestätigen und zu übernehmen.

Problem: Gast erscheint nicht in der Besetzung

Ursache: Nach Gast-Erstellung wurde das Cast-Management nicht gespeichert.

Lösung: Speichern Sie das Cast-Management mit "OK" um alle Änderungen einschließlich Gäste zu übernehmen.

Problem: Leerer Name wird akzeptiert

Ursache: System hat keine Validierung für leere Namen.

Lösung: Geben Sie immer einen aussagekräftigen Namen ein, um Verwirrung zu vermeiden.

Best Practices

🚀 Effizienz-Tipps

Konsistente Namenskonvention: Verwenden Sie einheitliche Formate wie "Vorname Nachname (Rolle)" für bessere Erkennbarkeit.

Kontakt-Information: Bei wichtigen Gästen erwägen Sie "Name (Tel: xxx)" für schnelle Erreichbarkeit.

Eindeutige Bezeichnungen: Vermeiden Sie generische Namen wie "Gast1" - nutzen Sie spezifische Bezeichnungen.

Rolle kennzeichnen: Fügen Sie die Rolle hinzu: "Max Mustermann (Fotograf)" statt nur "Max Mustermann".

Erweiterte Funktionen

Integration mit Cast-Management

Der Gast-Dialog ist nahtlos in das Cast-Management-System integriert:

  • Automatischer Aufruf - Wird direkt vom Cast-Management geöffnet bei "Gast"-Auswahl
  • Rückgabe-Integration - Name wird automatisch in das Cast-Management übernommen
  • UUID-System - Gäste erhalten spezielle UUID für System-interne Unterscheidung
  • Termin-spezifische Bindung - Gäste sind an den jeweiligen Termin gebunden

Datenhandling

Der Dialog behandelt Gast-Daten mit spezieller Logik:

  • String-basierte Speicherung - Gast-Namen werden als Text gespeichert, nicht als Mitarbeiter-Entität
  • Flexible Eingabe - Keine Validierung oder Formatierung der Namen
  • Direkte Übertragung - Name wird 1:1 vom Dialog ins Cast-System übernommen
  • Temporäre Existenz - Gäste existieren nur im Kontext des spezifischen Termins

Workflow-Integration

Der Gast-Dialog fügt sich optimal in den Planungsworkflow ein:

  • Unterbrechungsfreier Ablauf - Schnelle Gast-Eingabe ohne Workflow-Unterbrechung
  • Kontextuelle Verwendung - Wird nur geöffnet, wenn Gäste tatsächlich benötigt werden
  • Einfache Bedienung - Minimale Komplexität für maximale Effizienz
  • Sofortige Verfügbarkeit - Gast steht nach Bestätigung direkt für Zuweisung bereit

System-Architektur

Technische Aspekte der Gast-Funktionalität:

  • Lightweight Design - Minimaler Dialog ohne komplexe Datenstrukturen
  • Parent-Child Relationship - Korrekte Verknüpfung mit Cast-Management Dialog
  • Property Access - Gast-Name über le_guest.text() abrufbar
  • Standard Dialog-Verhalten - Verwendet Qt-Standard für OK/Cancel-Logik
