Notizen-System

Hilfe > Notizen-System

Übersicht

Das Notizen-System in HCC Plan ermöglicht die strukturierte Verwaltung von Informationen auf verschiedenen Planungsebenen. Von Team-weiten Hinweisen bis zu spezifischen Termin-Details können alle relevanten Informationen erfasst und automatisch weitergegeben werden.

Hauptfunktionen

  • Hierarchische Notizen-Struktur - Team → Planungsperiode → Event → Termin
  • Automatische Vererbung - Notizen werden auf untergeordnete Ebenen kopiert
  • Context-sensitive Dialoge - Angepasste Eingabeformulare je nach Notiz-Typ
  • Reset-Funktionalität - Wiederherstellung von übergeordneten Notizen
  • Export-Integration - Automatische Übernahme in generierte Zeitpläne und Excel-Dateien

Grundbedienung

  1. Notizen-Dialog über entsprechende Funktion öffnen (kontextabhängig):
    • Team-Notizen: Team-Verwaltung → "Notizen bearbeiten"
    • Plan-Notizen: Plan-Ansicht → "Plan-Informationen"
    • Event-Notizen: Event-Kontext → "Event-Notizen"
    • Termin-Notizen: Termin-Details → "Termin-Informationen"
  2. Relevante Informationen im Textfeld eingeben oder bearbeiten
  3. Optional Reset-Funktion nutzen um übergeordnete Notizen zu übernehmen
  4. Bei Plan-Notizen: Optional "Auch als Planungsperioden-Information speichern" aktivieren
  5. "OK" klicken zum Speichern oder "Abbrechen" zum Verwerfen

Tastaturkürzel

  • F1 - Diese Hilfe anzeigen
  • Ctrl+A - Gesamten Text auswählen
  • Ctrl+C/V - Kopieren und Einfügen
  • Enter - Neue Zeile im Textfeld
  • Ctrl+Enter - Speichern (OK)
  • Esc - Abbrechen ohne Speichern

Anwendungsbereich

Das Notizen-System wird verwendet für:

  • Team-weite Informationen - Allgemeine Hinweise, Kontaktdaten, Besonderheiten
  • Planungsperioden-Details - Spezifische Informationen für Zeitraum und Team-Kombination
  • Event-Koordination - Logistische Details, Anweisungen, besondere Anforderungen
  • Termin-spezifische Hinweise - Individuelle Notizen für Excel-Export
  • Kommunikation - Informationsweitergabe zwischen Planungsebenen

Notiz-Typen im Detail

Team-Notizen (DlgTeamNotes)

  • Zweck: Allgemeine Team-Informationen
  • Vererbung: Werden automatisch zu neuen Planungsperioden kopiert
  • Anwendung: Team-Kontakte, Standard-Anweisungen, wiederkehrende Hinweise
  • Zugriff: Über Team-Verwaltung

Plan-Notizen (DlgPlanPeriodNotes)

  • Zweck: Informationen für generierte Zeitpläne
  • Features: Reset von Planungsperiode + Rückspeichern-Option
  • Anwendung: Spezielle Anweisungen, Änderungen, wichtige Hinweise
  • Export: Erscheinen in generierten Zeitplan-Dokumenten

Event-Notizen (DlgEventNotes)

  • Zweck: Event-spezifische Informationen
  • Features: Unterstützt Mehrfach-Events-Modus
  • Vererbung: Werden zu verknüpften Terminen kopiert
  • Anwendung: Logistik-Details, Besonderheiten, Koordinations-Infos

Termin-Notizen (DlgAppointmentNotes)

  • Zweck: Individuelle Termin-Informationen
  • Features: Reset von Planning Event + spezielle Darstellung
  • Export: Werden in Excel-Dateien der Zeitpläne eingebunden
  • Anwendung: Personen-spezifische Hinweise, Last-Minute-Änderungen

Praktische Anwendungsfälle

💡 Informations-Kaskade

Effiziente Informationsweitergabe:

  1. Team-Ebene: "Kontakt Technik: Max Mustermann, Tel: 123-456"
  2. Plan-Ebene: "Besondere Anforderungen: Zusätzliche Sicherheit erforderlich"
  3. Event-Ebene: "Aufbau ab 8:00, Equipment-Check 9:30"
  4. Termin-Ebene: "Person A: Früh da für Setup, Person B: Nur Hauptevent"

Szenario 1 - Neues Projekt Setup:

  1. Team-Notizen mit Standard-Informationen anlegen
  2. Erste Planungsperiode erstellen → Team-Notizen werden übernommen
  3. Plan-spezifische Ergänzungen hinzufügen
  4. Events erstellen → Plan-Notizen als Basis nutzen
  5. Termin-Details individuell anpassen

Szenario 2 - Last-Minute-Änderungen:

  1. Termin-Notizen öffnen für betroffene Termine
  2. Änderung eingeben: "ACHTUNG: Raum geändert auf B-204"
  3. Speichern → Information erscheint in nächstem Excel-Export
  4. Bei Bedarf auch Event-Notizen aktualisieren für zukünftige Termine

Häufige Probleme & Lösungen

Problem: Notizen erscheinen nicht im Export

Ursache: Falsche Notiz-Ebene oder Export-Typ

Lösung: Plan-Notizen für Zeitplan-Generation, Termin-Notizen für Excel-Export verwenden. Prüfen Sie, ob die richtige Notiz-Ebene verwendet wird.

Problem: Team-Notizen werden nicht vererbt

Ursache: Planungsperiode bereits vor Team-Notizen erstellt

Lösung: Vererbung funktioniert nur bei NEU erstellten Planungsperioden. Bestehende Perioden manuell aktualisieren oder Reset-Funktion nutzen.

Problem: Text wird abgeschnitten

Ursache: Zu lange Notizen für bestimmte Export-Formate

Lösung: Notizen kompakt formulieren. Bei Excel-Export beachten, dass Zellengröße begrenzt ist. Wichtigste Infos an den Anfang setzen.

Problem: Reset-Funktion funktioniert nicht

Ursache: Übergeordnete Notiz ist leer oder nicht vorhanden

Lösung: Überprüfen Sie, ob auf der übergeordneten Ebene (Planungsperiode/Event) tatsächlich Notizen vorhanden sind.

Best Practices

🚀 Effizienz-Tipps

  • Hierarchie nutzen: Standard-Infos auf Team-Ebene, Spezifisches auf niedrigeren Ebenen
  • Konsistente Struktur: Ähnliche Informationen immer in gleicher Reihenfolge
  • Kurz und prägnant: Wichtigste Infos zuerst, Details optional
  • Reset-Funktionen nutzen: Übergeordnete Notizen als Ausgangsbasis verwenden
  • Export-Format bedenken: Notizen entsprechend der Ausgabe-Ziel formatieren

⚡ Content-Struktur

Empfohlenes Format für Notizen:

  • Team-Notizen: Kontakte | Standards | Besonderheiten
  • Plan-Notizen: Änderungen | Wichtige Termine | Ansprechpartner
  • Event-Notizen: Zeit/Ort Details | Equipment | Logistik
  • Termin-Notizen: Person-spezifisch | Ausnahmen | Erinnerungen

Erweiterte Funktionen

Mehrfach-Events-Modus

Der Event-Notizen-Dialog kann im Mehrfach-Modus verwendet werden, um identische Notizen für mehrere Events gleichzeitig zu setzen. Dies ist besonders nützlich bei wiederkehrenden Events oder Event-Serien.

Bidirektionale Speicherung

Plan-Notizen bieten die Option "Auch als Planungsperioden-Information speichern", wodurch Änderungen sowohl für den spezifischen Plan als auch für die zugrunde liegende Planungsperiode gespeichert werden.

Context-Aware Display

Jeder Dialog zeigt relevante Kontext-Informationen (Datum, Zeit, Ort, Team) automatisch an, um die richtige Zuordnung der Notizen sicherzustellen.

Technische Details

Das Notizen-System basiert auf vier spezialisierten Dialog-Klassen, die jeweils mit entsprechenden Datenbank-Schemas arbeiten. Die automatische Vererbung erfolgt über Datenbank-Triggers und Cascade-Funktionen. Notizen werden als Plain-Text gespeichert und bei Export-Funktionen entsprechend formatiert. Die Reset-Funktionalität greift auf Parent-Objekte zu und aktualisiert den UI-State entsprechend.
