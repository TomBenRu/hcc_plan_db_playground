Partner-Standort-Präferenzen

Hilfe > Partner-Standort-Präferenzen

Übersicht

Der Dialog "Partner Preferences" ist das zentrale System zur Konfiguration von Mitarbeiter-Partner- und Standort-Präferenzen in HCC Plan. Dieses sophisticated Interface ermöglicht die fein-granulare Steuerung von Zuweisungsprioritäten über ein 5-stufiges Slider-System (0-4), kombiniert mit intelligenter Team- und Datums-Integration. Das System unterstützt sowohl globale Präferenz-Management als auch detaillierte Drill-Down-Dialoge für spezifische Partner-Standort-Kombinationen.

Hauptfunktionen

  • Dual-Group-Interface - Getrennte Scroll-Areas für Mitarbeiter- und Standort-Präferenzen
  • 5-Stufiges-Bewertungssystem - Slider von "nicht zuteilen" (0) bis "zwingend zuteilen" (4)
  • Datumsbasierte Team-Kalkulation - Dynamische Partner/Standort-Listen basierend auf gewähltem Datum
  • Drill-Down-Dialoge - Detaillierte Einzelkonfiguration über "Employees"/"Facilities"-Buttons
  • Intelligente Score-Aggregation - Automatische Berechnung von Maximum-Werten bei Multi-Partner-Kombinationen
  • Parent-Model-Integration - Vererbung und Reset-Optionen von übergeordneten Modellen
  • Command-Pattern-Undo/Redo - Vollständige Rückgängigmachung komplexer Präferenz-Änderungen
  • Adaptive UI-Sizing - Automatische Scroll-Area-Anpassung an Screen-Geometrie
  • Real-time Validation - Sofortige Aktualisierung aller abhängigen UI-Elemente
  • Cleanup-Automatisierung - Automatisches Entfernen ungenutzter Präferenz-Einträge

Grundbedienung

  1. Dialog öffnen - Zugriff über Mitarbeiter-Management oder Cast-Konfiguration
  2. Datum für Team-Berechnung wählen:
    • Setzen Sie das Datum über das DateEdit-Widget
    • System berechnet automatisch verfügbare Partner und Standorte für dieses Datum
    • Info-Label zeigt Team-Konsistenz über Planungsperiode an
  3. Globale Präferenzen mit Slidern konfigurieren:
    • Mitarbeiter-Sektion: Slider für jeden Partner setzen (0-4 Skala)
    • Standort-Sektion: Slider für jeden Standort setzen (0-4 Skala)
    • Label zeigt Slider-Wert als Text ("nicht zuteilen", "gerne zuteilen" etc.)
  4. Detaillierte Konfiguration über Drill-Down:
    • "Employees" Button: Öffnet standort-spezifische Partner-Präferenzen
    • "Facilities" Button: Öffnet partner-spezifische Standort-Präferenzen
    • Button-Farben zeigen Konsistenz-Status (alle gleich vs. gemischt)
  5. Reset-Optionen nutzen:
    • Einzelnes Parent-Model: "Reset" setzt alle Werte auf "Normal" (2)
    • Parent-Model vorhanden: Dropdown mit "Reset to Normal" und "Values from parent model"
  6. Änderungen speichern:
    • "Save" - Speichert alle Änderungen und entfernt ungenutzte Einträge
    • "Cancel" - Verwirft alle Änderungen über Undo-System

Slider-System und Bewertungsskala

5-Stufige Bewertungsskala

💡 Score-System (0-4)

0 - "nicht zuteilen": Partner/Standort-Kombination wird aktiv vermieden

1 - "falls nötig zuteilen": Nur bei Personalmangel oder Notfall verwenden

2 - "gerne zuteilen": Standard-Wert, neutrale Präferenz ohne Bevorzugung

3 - "bevorzugt zuteilen": Bevorzugte Kombination, wird favorisiert

4 - "zwingend zuteilen": Optimale Kombination, höchste Priorität

Score-Berechnung und Aggregation

  • Internal Storage - Scores werden als 0.0-2.0 float in Database gespeichert (value/2)
  • UI Display - Anzeige als 0-4 Integer für bessere Benutzerfreundlichkeit
  • Default-Handling - Fehlende Präferenzen werden als Score 1 ("gerne zuteilen") behandelt
  • Maximum-Aggregation - Bei Multi-Partner-Slidern wird höchster Einzelwert angezeigt

Praktische Anwendungsfälle

🎯 Optimierungs-Strategien

Häufige Probleme & Lösungen

Problem: Partner/Standorte werden nicht angezeigt

Ursache: Gewähltes Datum liegt außerhalb gültiger Team-Zuordnungen oder Person ist an diesem Tag nicht aktiv.

Lösung: Wählen Sie ein anderes Datum innerhalb der aktiven Planungsperiode der Person.

Problem: "Not all days have same partners/facilities" Info-Meldung

Ursache: Team-Zusammensetzung ändert sich über die Planungsperiode, Union aller Optionen wird angezeigt.

Lösung: Normal - System zeigt alle möglichen Kombinationen zur umfassenden Konfiguration.

Problem: Slider-Änderungen werden nicht gespeichert

Ursache: Updating_sliders-Flag verhindert rekursive Updates während automatischer Berechnungen.

Lösung: Warten Sie bis UI-Updates abgeschlossen sind, dann erneut versuchen.

Problem: Reset-Button hat keine Wirkung

Ursache: Bei Parent-Model ist Reset ein Dropdown-Menu, kein direkter Button-Click.

Lösung: Klicken Sie auf Pfeil neben Reset-Button und wählen gewünschte Reset-Option.

Problem: Dialog wird zu groß für Bildschirm

Ursache: Viele Partner/Standorte führen zu großen Scroll-Areas.

Lösung: System repositioniert Dialog automatisch, Scroll-Areas haben Maximum-Height-Limits.

Best Practices

🚀 Effizienz-Optimierung

Strategic Scoring: Verwenden Sie Extremwerte (0,4) sparsam - meist reichen 1-3 für effektive Optimierung.

Drill-Down-Efficiency: Nutzen Sie globale Slider für Basis-Konfiguration, Drill-Downs nur für Feintuning.

Parent-Model-Leverage: Bei ähnlichen Rollen nutzen Sie Parent-Model-Vererbung statt kompletter Neukonfiguration.

Validation-Driven: Beachten Sie Button-Farben zur Identifikation von Inkonsistenzen die Aufmerksamkeit brauchen.

🎯 Qualitäts-Management

Balanced Approach: Vermeiden Sie zu restriktive Konfigurationen - System braucht Flexibilität für optimale Zuweisungen.

Regular Reviews: Überprüfen Sie Präferenzen regelmäßig und anpassen basierend auf realer Performance.

Team Feedback: Integrieren Sie Mitarbeiter-Feedback in Präferenz-Konfiguration für bessere Akzeptanz.

Documentation: Dokumentieren Sie Gründe für Extremwerte (0,4) für spätere Nachvollziehbarkeit.
