Planungsperioden-Verwaltung

Hilfe > Planungsperioden-Verwaltung

Übersicht

Die "Planungsperioden-Verwaltung" umfasst sowohl das Erstellen neuer als auch das Bearbeiten bestehender Planungszeiträume für Teams. Eine Planungsperiode ist der grundlegende Zeitrahmen, in dem Termine, Verfügbarkeiten und Ressourcen für ein Team geplant werden.

Verfügbare Modi

  • Erstellen-Modus - Neue Planungsperioden definieren und automatisch mit Server synchronisieren
  • Bearbeiten-Modus - Bestehende Planungsperioden auswählen, ändern und löschen

Hauptfunktionen

Gemeinsame Features (Erstellen + Bearbeiten)

  • Team-basierte Planung - Zuweisung von Planungsperioden zu spezifischen Teams
  • Planer-Auswahl - Bestimmung des verantwortlichen Planers für die Periode
  • Flexible Zeitraumdefinition - Start-, End- und Deadline-Festlegung mit automatischer Validierung
  • Duale Notizenfunktion - Interne Notizen und separate Notizen für das Online-Portal
  • Erinnerungssystem - Optional automatische Benachrichtigungen

Erstellen-spezifische Features

  • Server-Synchronisation - Automatische Übertragung zum Online-Portal mit Fallback-Option
  • Automatische Ressourcenerstellung - Locations und Akteure werden basierend auf Teamkonfiguration erstellt

Bearbeiten-spezifische Features

  • Planungsperioden-Auswahl - Dropdown zur Auswahl bestehender Perioden
  • Lösch-Funktion - Bestehende Planungsperioden entfernen
  • Team-Notizen kopieren - Button zum Zurücksetzen der Notizen aus Team-Einstellungen
  • Intelligent Constraints - Datumsgrenzen basierend auf benachbarten Perioden

Grundbedienung

Planungsperiode Erstellen

  1. Planer auswählen - Wählen Sie den verantwortlichen Planer aus der "Planer"-Dropdown-Liste
  2. Team festlegen - Bestimmen Sie das Team aus der "Team"-Auswahl (wird automatisch nach Planer-Auswahl gefiltert)
  3. Zeitraum definieren:
    • Setzen Sie das "Start"-Datum der Planungsperiode
    • Definieren Sie das "End"-Datum der Planungsperiode
    • Bestimmen Sie die "Deadline" für die Terminplanung
  4. Notizen hinzufügen:
    • Tragen Sie interne "Notizen" für die Planungsverwaltung ein
    • Verfassen Sie "Notizen im Online Portal" für Mitarbeiter-Sichtbarkeit
  5. Erinnerung konfigurieren - Aktivieren Sie optional "Erinnerung senden?"
  6. Speichern - Klicken Sie auf "Speichern" zum Erstellen der Planungsperiode

Planungsperiode Bearbeiten

  1. Planer auswählen - Wählen Sie den Planer aus der "Planer auswählen"-Dropdown-Liste
  2. Team bestimmen - Wählen Sie das gewünschte Team aus "Team auswählen"
  3. Planungsperiode wählen - Selektieren Sie die zu bearbeitende Periode aus "Planung auswählen"
  4. Daten anpassen:
    • Ändern Sie "Start", "Ende" und "Deadline" nach Bedarf
    • Beachten Sie die automatischen Datumsgrenzen basierend auf benachbarten Perioden
  5. Notizen aktualisieren:
    • Bearbeiten Sie die "Notizen" oder nutzen Sie "Aus Team-Notizen kopieren"
    • Aktualisieren Sie "API-Nachrichten an Mitarbeiter"
  6. Aktion wählen:
    • "Speichern" - Änderungen übernehmen
    • "Löschen" - Planungsperiode entfernen

Tastaturkürzel

F1 - Diese Hilfe öffnen

Enter - Planungsperiode speichern (wenn alle Pflichtfelder ausgefüllt)

Escape - Dialog abbrechen

Tab - Zwischen Eingabefeldern wechseln

Verwandte Funktionen

  • Planungsperiode Bearbeiten - Bestehende Planungsperioden ändern
  • Planungsübersicht - Hauptplanungsansicht mit Terminen und Verfügbarkeiten
  • Team-Verwaltung - Teams und Planer konfigurieren

Anwendungsbereich

🎯 Die Planungsperioden-Erstellung wird verwendet für:

  • Neue Planungszyklen - Start eines neuen Planungszeitraums für ein Team
  • Saisonale Planung - Definition spezieller Zeiträume (z.B. Urlaubszeiten, Feiertage)
  • Projekt-spezifische Perioden - Zeiträume für besondere Projekte oder Events
  • Team-Reorganisation - Nach Änderungen in der Teamstruktur

Benutzeroberfläche

Gemeinsame UI-Elemente

  • Planer-Dropdown - Zeigt alle berechtigten Planer mit Icons an
  • Team-Dropdown - Dynamisch gefiltert basierend auf Planer-Auswahl
  • Datum-Eingaben - Start, Ende und Deadline mit Kalenderpopup
  • Notizen-Textfelder - Mehrzeilige Eingabe für interne und Portal-Notizen
  • Erinnerungs-Checkbox - Aktivierung automatischer Benachrichtigungen

Bearbeiten-spezifische UI-Elemente

  • Planungsperioden-Dropdown - Auswahl bestehender Perioden (chronologisch sortiert)
  • GroupBox "Planung auswählen" - Organisiert die Auswahlfelder für Planer/Team/Periode
  • GroupBox "Planungsdaten" - Enthält alle Eingabefelder für die Bearbeitung
  • "Aus Team-Notizen kopieren" Button - Setzt Notizen auf Team-Standard zurück
  • "Löschen" Button - Entfernt die ausgewählte Planungsperiode

Automatische Validierungen

  • Datumslogik - Start-Datum darf nicht nach End-Datum liegen
  • Periodenfolge - Neue Periode muss nach der letzten bestehenden Periode beginnen
  • Team-Pflicht - Ein Team muss ausgewählt werden vor dem Speichern

Praktische Anwendungsfälle

💡 Häufige Szenarien

Monatliche Planung: Erstellen Sie Planungsperioden für jeden Monat mit einer Woche Vorlaufzeit für die Deadline.

Urlaubsperioden: Definieren Sie spezielle Perioden für Ferienszeiten mit angepassten Notizen für das Online-Portal.

Projekt-Phasen: Nutzen Sie separate Planungsperioden für verschiedene Projektabschnitte mit entsprechenden internen Notizen.

Häufige Probleme & Lösungen

Problem: "Sie müssen zuerst ein Team auswählen" Fehlermeldung

Ursache: Kein Team wurde vor dem Speichern ausgewählt.

Lösung: Wählen Sie einen Planer aus, dann automatisch ein Team, bevor Sie speichern.

Problem: Datum kann nicht gesetzt werden

Ursache: Das gewählte Datum liegt vor der letzten bestehenden Planungsperiode.

Lösung: Wählen Sie ein Start-Datum nach dem Ende der letzten Planungsperiode des Teams.

Problem: Server-Synchronisation fehlgeschlagen (nur Erstellen)

Ursache: Keine Internetverbindung oder Server-Problem.

Lösung: Wählen Sie "Ja" für lokale Erstellung, synchronisieren Sie später manuell.

Problem: Eingabefelder sind deaktiviert (nur Bearbeiten)

Ursache: Keine Planungsperiode ausgewählt oder keine Perioden für das gewählte Team vorhanden.

Lösung: Wählen Sie eine gültige Planungsperiode aus der Dropdown-Liste aus.

Problem: Datum lässt sich nicht ändern (nur Bearbeiten)

Ursache: Datumsgrenzen durch benachbarte Planungsperioden eingeschränkt.

Lösung: Beachten Sie die Min/Max-Grenzen - Start muss nach vorheriger Periode, Ende vor nachfolgender Periode liegen.

Best Practices

🚀 Effizienz-Tipps

Konsistente Zeiträume: Verwenden Sie gleichmäßige Planungsperioden (z.B. immer 4 Wochen) für bessere Übersichtlichkeit.

Ausreichende Vorlaufzeit: Setzen Sie die Deadline mindestens eine Woche vor dem Start für rechtzeitige Planung.

Klare Portal-Notizen: Schreiben Sie Notizen für das Online-Portal verständlich für alle Mitarbeiter.

Erinnerungen nutzen: Aktivieren Sie Erinnerungen für kritische Planungsperioden.

Vorsicht beim Bearbeiten: Änderungen an Start/Ende können bestehende Termine beeinflussen - prüfen Sie die Planungsübersicht.

Team-Notizen nutzen: Verwenden Sie "Aus Team-Notizen kopieren" um konsistente Standardnotizen zu gewährleisten.

Erweiterte Funktionen

Automatische Ressourcenerstellung

Nach erfolgreicher Erstellung werden automatisch folgende Objekte generiert:

  • Location-Plan-Perioden - Für alle dem Team zugewiesenen Standorte
  • Actor-Plan-Perioden - Für alle Teammitglieder
  • Event-Gruppen - Für Location-basierte Terminplanung
  • Verfügbarkeits-Gruppen - Für Actor-basierte Verfügbarkeitsplanung

Server-Integration (nur Erstellen)

Neu erstellte Planungsperioden werden automatisch an das Online-Portal übertragen:

  • Erfolgs-Bestätigung - Detaillierte Information über die erstellte Periode
  • Fehler-Behandlung - Fallback-Option für lokale Erstellung bei Server-Problemen
  • Rollback-Möglichkeit - Rückgängigmachung bei Übertragungsfehlern

Edit-spezifische Features

Beim Bearbeiten bestehender Planungsperioden:

  • Intelligente Datumsgrenzen - Min/Max-Werte basierend auf benachbarten Perioden
  • Lokale Änderungen - Bearbeitungen werden nur lokal gespeichert (keine automatische Server-Synchronisation)
  • Lösch-Sicherheit - Lösch-Button separat von Speichern/Abbrechen positioniert
  • Team-Integration - Direkte Übernahme von Team-Notizen möglich
