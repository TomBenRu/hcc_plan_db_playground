Skill-Gruppen

Hilfe > Skill-Gruppen

Übersicht

Skill-Gruppen definieren Personalanforderungen für Orte und Events. Im Gegensatz zu individuellen Skills, die Fähigkeiten einzelner Personen beschreiben, legen Skill-Gruppen fest, wie viele Personen mit welchen Qualifikationen für bestimmte Locations oder Events benötigt werden.

Hauptfunktionen

  • Anforderungs-Definition - Kombination aus Skill + benötigte Personenanzahl
  • Location-basierte Gruppen - Skill-Anforderungen für Arbeitsorte
  • Event-basierte Gruppen - Spezifische Anforderungen für einzelne Events
  • Personalstärke-Planung - 1-100 Personen pro Skill definierbar
  • Reset-Funktion - Event-Skills auf Location-Standard zurücksetzen
  • Intelligente Skill-Auswahl - Automatische Filterung bereits verwendeter Skills

Konzeptuelle Abgrenzung

🎯 Skills vs. Skill-Gruppen

Skills: "Person X kann schauspielern" (individuelle Fähigkeiten)
Skill-Gruppen: "Wir brauchen 3 Schauspieler für diese Location" (Personalanforderungen)

Die zwei Dialog-Typen

1. Skill-Gruppen-Verwaltung (Haupt-Dialog)

Zweck: Verwaltung aller Skill-Gruppen für eine Location oder ein Event

  • Skill-Gruppen-Tabelle - Zeigt Skill, Beschreibung und Personenanzahl
  • New Skill Group - Erstellt neue Anforderungsgruppen
  • Edit Skill Group - Bearbeitet bestehende Gruppen
  • Remove Skill Group - Entfernt Anforderungen
  • Reset - Nur bei Events: Zurück zu Location-Standard
  • Kontext-sensitive Anzeige - Unterschiedlich für Locations vs. Events

2. Skill-Gruppe bearbeiten (Einzelne Gruppe)

Zweck: Definition einer einzelnen Skill-Anforderung

  • Skill-Auswahl - Dropdown mit allen verfügbaren Projekt-Skills
  • Personenanzahl - Spinner für 1-100 benötigte Personen
  • Intelligente Filterung - Bereits verwendete Skills ausgeblendet
  • Dual-Modus - Für neue und bestehende Skill-Gruppen

Grundbedienung

Skill-Gruppen für eine Location definieren

  1. Öffnen Sie den Skill Groups-Dialog für die gewünschte Location
  2. Klicken Sie New Skill Group
  3. Wählen Sie den benötigten Skill aus dem Dropdown
  4. Stellen Sie die Anzahl Personen ein (1-100)
  5. Bestätigen Sie mit OK
  6. Die neue Skill-Gruppe erscheint in der Tabelle

Event-spezifische Anpassungen

  1. Öffnen Sie die Skill-Gruppen für ein spezifisches Event
  2. Das System zeigt zunächst die Standard-Anforderungen der Location
  3. Passen Sie Gruppen für dieses Event an (Add/Edit/Remove)
  4. Nutzen Sie Reset um zu Location-Standard zurückzukehren

Bestehende Skill-Gruppen bearbeiten

  1. Wählen Sie eine Skill-Gruppe in der Tabelle aus
  2. Doppelklicken oder Edit Skill Group verwenden
  3. Ändern Sie Skill oder Personenanzahl
  4. Bestätigen Sie die Änderungen

Verwandte Funktionen

  • Individual Skills - Für die Definition der grundlegenden Skills, die in Gruppen verwendet werden
  • Stammdaten - Für Location-Verwaltung und Event-Erstellung
  • Hauptplanung - Für die Verwendung der Skill-Anforderungen in der Planungslogik

Anwendungsbereich

Location-basierte Skill-Gruppen

  • Theaterbühne: "3 Schauspieler, 2 Techniker, 1 Regisseur"
  • Werkstatt: "2 Kostümschneider, 1 Requisiteur"
  • Proberaum: "4 Musiker, 1 Dirigent"

Event-spezifische Anpassungen

  • Premiere: Zusätzliche Techniker für komplexere Technik
  • Kindervorstellung: Weniger Schauspieler, zusätzliche Betreuer
  • Proben: Reduzierte Personalstärke im Vergleich zu Aufführungen

Praktische Anwendungsfälle

🎯 Szenario 1: Neue Location - Standard-Anforderungen

Für eine neue Probebühne werden die Standard-Anforderungen definiert: "2 Schauspieler", "1 Regisseur", "1 Techniker". Diese dienen als Basis für alle Events an diesem Ort.

🎯 Szenario 2: Spezial-Event - Angepasste Anforderungen

Für eine Gala-Aufführung werden die Standard-Anforderungen einer Location überschrieben: Statt 2 Schauspieler werden 5 benötigt, zusätzlich 2 Extra-Techniker.

🎯 Szenario 3: Proben vs. Aufführung

Proben-Events erhalten reduzierte Skill-Gruppen (1 statt 2 Techniker), während Aufführungs-Events die vollen Location-Anforderungen nutzen.

🎯 Szenario 4: Personalplanung

Das Planungssystem verwendet Skill-Gruppen, um automatisch zu berechnen, wie viele Personen mit welchen Qualifikationen für verschiedene Termine benötigt werden.

Häufige Probleme & Lösungen

Problem: Skill nicht in Auswahl verfügbar

Ursache: Skill ist bereits als Skill-Gruppe für diese Location/Event definiert

Lösung: Jeder Skill kann nur einmal pro Location/Event als Gruppe verwendet werden

Problem: Reset-Button nicht verfügbar

Ursache: Reset-Funktion ist nur für Events verfügbar, nicht für Locations

Lösung: Locations definieren die Standard-Anforderungen - nur Events können auf diese zurückgesetzt werden

Problem: Keine Skills zur Auswahl

Ursache: Alle verfügbaren Projekt-Skills sind bereits als Gruppen verwendet

Lösung: Erstellen Sie zuerst neue Skills über die Individual Skills Verwaltung

Best Practices

💡 Location-First Approach

Definieren Sie zunächst realistische Standard-Anforderungen für Locations. Event-spezifische Anpassungen sollten die Ausnahme sein, nicht die Regel.

🚀 Realistische Personenzahlen

Berücksichtigen Sie die verfügbaren Ressourcen bei der Definition von Skill-Gruppen. Zu hohe Anforderungen können zu unlösbaren Planungsproblemen führen.

⚡ Event-Vererbung nutzen

Events erben automatisch die Skill-Gruppen ihrer Location. Nutzen Sie dies für konsistente Grundanforderungen und passen Sie nur bei Bedarf an.

🔄 Planungshierarchie verstehen

Skills (Grundlage) → Skill-Gruppen (Anforderungen) → Planung (Automatische Zuweisung). Jede Ebene baut auf der vorherigen auf.

Workflow-Integration

🔄 Planungsvorbereitungs-Workflow

  1. Skills definieren: Über Individual Skills alle benötigten Qualifikationen erstellen
  2. Location-Anforderungen: Standard Skill-Gruppen für jeden Arbeitsort definieren
  3. Event-Anpassungen: Bei Bedarf spezifische Events mit angepassten Anforderungen
  4. Automatische Planung: System nutzt Skill-Gruppen für optimale Personalzuteilung

🔄 Location vs. Event Unterscheidung

  • Location-Skill-Gruppen: Permanente Standard-Anforderungen
  • Event-Skill-Gruppen: Temporäre, event-spezifische Anpassungen
  • Vererbung: Events starten mit Location-Standards, können aber überschrieben werden

Erweiterte Funktionen

Unterschiedliche Update-Logik

Locations: Direkte Update-Commands für permanente Änderungen
Events: Remove-Create-Add Sequenz für temporäre Anpassungen

Automatische Filterung

Das System verhindert automatisch Doppel-Definitionen: Jeder Skill kann nur einmal pro Location/Event als Gruppe verwendet werden.

Kaskaden-Reset für Events

Die Reset-Funktion entfernt alle event-spezifischen Skill-Gruppen und lädt die Standard-Gruppen der zugehörigen Location neu.

Tastaturkürzel

  • F1 - Diese Hilfe öffnen
  • Doppelklick - Skill-Gruppe bearbeiten (in Tabelle)
  • Enter - Aktion bestätigen
  • Escape - Dialog ohne Änderungen schließen
  • Tab - Zwischen Elementen wechseln
  • Del - Ausgewählte Skill-Gruppe entfernen
